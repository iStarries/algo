## AcWing 800. 数组元素的目标和 

`难度：简单`

### 题目描述

给定两个升序排序的有序数组A和B，以及一个目标值x。数组下标从0开始。
请你求出满足A[i] + B[j] = x的数对(i, j)。

数据保证有唯一解。

**输入格式**

第一行包含三个整数n，m，x，分别表示A的长度，B的长度以及目标值x。

第二行包含n个整数，表示数组A。

第三行包含m个整数，表示数组B。

**输出格式**

共一行，包含两个整数 i 和 j。

```r
数据范围

数组长度不超过100000。
同一数组内元素各不相同。
1≤数组元素≤109

输入样例：

4 5 6
1 2 4 7
3 4 6 8 9

输出样例：

1 1
```

### Solution

```java
import java.util.*;
import java.io.*;

public class Main{
    public static void main(String[] args) throws IOException{
        BufferedReader in = new BufferedReader(new InputStreamReader(System.in));
        String[] s = in.readLine().split(" ");
        int n = Integer.parseInt(s[0]);
        int m = Integer.parseInt(s[1]);
        int x = Integer.parseInt(s[2]);
        int[] a = new int[n + 10];
        int[] b = new int[m + 10];
        s = in.readLine().split(" ");
        for(int i = 0; i < n; i++){
            a[i] = Integer.parseInt(s[i]);
        }
        s = in.readLine().split(" ");
        for(int i = 0; i < m; i++){
            b[i] = Integer.parseInt(s[i]);
        }
        // A 数组正序遍历
        // B 数组倒序遍历
        for(int i = 0, j = m - 1; i < n; i++){
            while(j > 0 && a[i] + b[j] > x) j--;
            if(a[i] + b[j] == x){
                System.out.println(i + " " + j);
            }
        }
        
    }
}
```

## 一、题型与核心考点

### 这题是什么类型？

给你两个**升序数组** `A` 和 `B`，要找一组下标 `(i, j)`，满足：

```text
A[i] + B[j] = x
```

并且题目保证**唯一解**。

### 核心考点

这题的关键不是“枚举答案”，而是利用：

* **数组有序**
* **和太大时要减小**
* **和太小时要增大**

这正是双指针能发挥作用的信号。

---

## 二、先看朴素做法，再看优化做法

### 1）朴素做法：两层循环枚举

最直接的想法是：

* 枚举 `i`
* 再枚举 `j`
* 判断 `A[i] + B[j] == x`

时间复杂度是：

```text
O(n * m)
```

而题目里 `n, m` 最多到 `100000`，这样最坏会到 `10^10` 次比较，肯定超时。

---

### 2）优化做法：双指针

因为两个数组都是升序，所以我们可以这样想：

* `i` 从 `A` 的开头开始走，也就是从小到大
* `j` 从 `B` 的结尾开始走，也就是从大到小

这样：

* 如果 `A[i] + B[j] > x`，说明和太大了，需要让它变小
  那就把 `j--`
* 如果 `A[i] + B[j] < x`，说明和太小了，需要让它变大
  那就把 `i++`
* 如果相等，就找到答案

这个过程里，`i` 只会往右走，`j` 只会往左走，所以总共最多移动 `n + m` 次。

时间复杂度就变成：

```text
O(n + m)
```

这才是这题真正想考的做法。

---

## 三、为什么这样走一定不会漏答案？

这是初学者最容易疑惑的地方。

假设当前在 `(i, j)`：

### 情况1：`A[i] + B[j] > x`

因为 `B` 是升序数组，`j` 右边的数更大，左边的数更小。
此时和已经太大了，继续增大 `i` 只会让和更大，所以**不可能通过增大 `i` 来得到答案**。

因此这时唯一合理的方向就是：

```text
j--
```

---

### 情况2：`A[i] + B[j] < x`

同理，和太小了。
如果你减小 `j`，那么 `B[j]` 会更小，和会更小，更不可能等于 `x`。
所以这时必须：

```text
i++
```

---

这就说明：
**每一步移动都不是“拍脑袋”，而是由有序性严格推出的。**

---

## 四、结合样例走一遍

题目样例：

```text
A = [1, 2, 4, 7]
B = [3, 4, 6, 8, 9]
x = 6
```

初始：

* `i = 0`，`A[i] = 1`
* `j = 4`，`B[j] = 9`

### 第1步

`1 + 9 = 10 > 6`
和太大，`j--`

### 第2步

`1 + 8 = 9 > 6`
还是太大，`j--`

### 第3步

`1 + 6 = 7 > 6`
还是太大，`j--`

### 第4步

`1 + 4 = 5 < 6`
和太小，`i++`

### 第5步

`2 + 4 = 6`
找到答案：`(1, 1)`

## 五、你这份题解代码在做什么？

你上传的代码核心是这段：

```java
for(int i = 0, j = m - 1; i < n; i++){
    while(j > 0 && a[i] + b[j] > x) j--;
    if(a[i] + b[j] == x){
        System.out.println(i + " " + j);
    }
}
```

### 它的含义

* `i` 从左到右扫 `A`
* `j` 从右到左缩 `B`
* 对于每个 `i`，只要当前和大于 `x`，就不断让 `j--`
* 因为 `j` 不会回头，所以整体仍然是线性的

这是标准双指针思想。

## 六、代码实现细节，逐行讲清楚

下面我给你一个更适合初学者理解、也更规范一点的 Java 写法：

```java
import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;

public class Main {
    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

        String[] first = br.readLine().split(" ");
        int n = Integer.parseInt(first[0]);
        int m = Integer.parseInt(first[1]);
        int x = Integer.parseInt(first[2]);

        int[] a = new int[n];
        int[] b = new int[m];

        String[] sa = br.readLine().split(" ");
        for (int i = 0; i < n; i++) {
            a[i] = Integer.parseInt(sa[i]);
        }

        String[] sb = br.readLine().split(" ");
        for (int i = 0; i < m; i++) {
            b[i] = Integer.parseInt(sb[i]);
        }

        int j = m - 1; // B数组从右端开始

        for (int i = 0; i < n; i++) {
            // 如果当前和太大，就让 j 左移
            while (j >= 0 && a[i] + b[j] > x) {
                j--;
            }

            // 这里题目保证有唯一解，所以最终一定能找到
            if (j >= 0 && a[i] + b[j] == x) {
                System.out.println(i + " " + j);
                break;
            }
        }
    }
}
```

## 七、这份代码里每个变量是什么意思？

### `n, m, x`

* `n`：数组 `A` 的长度
* `m`：数组 `B` 的长度
* `x`：目标和

### `a[]`, `b[]`

存两个升序数组。

### `i`

指向 `A` 当前考虑的位置，从左往右走。

### `j`

指向 `B` 当前考虑的位置，从右往左走。

---

## 八、为什么 `j` 不需要每次重新回到 `m - 1`？

这是这题最关键的优化点。

很多同学会写成：

* 固定一个 `i`
* 然后 `j` 从 `m - 1` 重新开始找

这样虽然也能做，但最坏还是接近 `O(nm)`。

而正确的双指针做法里：

* `i` 递增
* `A[i]` 会越来越大
* 因此为了让和不超过 `x`，`j` 只可能继续往左，不可能往右退回去

也就是说，`j` 的移动是“单调的”。

这就是复杂度能压到 `O(n + m)`` 的根本原因。

## 九、你这份原代码里有哪些细节值得注意？

你上传的解法总体思路是对的，但有几个点你可以留意一下。

### 1）`int[] a = new int[n + 10]`

你写了 `n + 10`、`m + 10`，这在竞赛里很常见，意思是“多开一点防止边界麻烦”。
但这题其实直接开成：

```java
int[] a = new int[n];
int[] b = new int[m];
```

就够了，更清晰。

---

### 2）`while (j > 0 && a[i] + b[j] > x) j--;`

这里写成 `j > 0` 是可以过很多测试的，但更严谨的思路常常会写成：

```java
while (j >= 0 && a[i] + b[j] > x) j--;
```

然后判断时再先检查 `j >= 0`。

这样边界更完整。

---

### 3）找到答案后最好 `break`

因为题目说**唯一解**，所以找到以后可以直接结束：

```java
break;
```

这样逻辑更干净。

## 十、常见错误

### 错误1：把双指针写回了双重循环

比如每次 `i` 变化后，让 `j` 重新从末尾开始。
这样就丢掉了“双指针单调移动”的精髓。

---

### 错误2：方向搞反

这题正确的搭配通常是：

* `i` 从左到右
* `j` 从右到左

因为这样一个负责增大和，一个负责减小和。

当然反过来也能设计，但初学阶段建议牢牢记住这一版。

---

### 错误3：边界判断不严谨

例如：

* `j` 减到 `-1` 还访问 `b[j]`
* 找到答案后不退出
* 数组读入下标写错

这些都是 Java 里常见的小失误。

---

## 十一、复杂度分析

### 时间复杂度

```text
O(n + m)
```

因为 `i` 最多走 `n` 次，`j` 最多走 `m` 次。

### 空间复杂度

```text
O(n + m)
```

主要是存数组。

如果只看算法额外空间（不算输入数组本身），那双指针本身是：

```text
O(1)
```

---

## 十二、可替代做法：哈希表

还有一种思路：

* 把 `B` 中每个值存到哈希表里，记录下标
* 枚举 `A[i]`
* 查找 `x - A[i]` 是否在 `B` 中

这样时间复杂度也是：

```text
O(n + m)
```

但空间复杂度会变成：

```text
O(m)
```

### 那为什么这题更推荐双指针？

因为题目已经给你了一个非常强的条件：**两个数组都是升序的**。
有序性一旦出现，双指针通常就是更自然、更省空间的选择。

## 十三、最后再讲：为什么能想到这种解法？

这个问题不要一开始想，放到最后总结最有帮助。

当你以后做题时，看到下面这些信号，就应该联想到双指针：

### 触发信号1：数组有序

题目明确说了两个数组都是升序。
“有序”几乎就是在提醒你：别暴力枚举，看看能不能利用单调性。

### 触发信号2：要求找两个位置满足某种关系

这里是：

```text
A[i] + B[j] = x
```

这是典型的“两个指针配合移动”的场景。

### 触发信号3：当前状态可以决定下一步往哪边走

* 和太大，就减小
* 和太小，就增大

只要出现这种“根据比较结果决定移动方向”的特征，双指针往往就很合适。

---

## 十四、方法迁移：以后看到哪些题也要想到它？

这类思路可以迁移到很多题：

### 1）一个有序数组里找两数之和

最经典的左右指针夹逼。

### 2）两个有序数组求交集

两个指针一起走，小的往前移动。

### 3）归并过程

本质上也是双指针。

### 4）删除/保留某些元素

像“快慢指针”也是双指针的另一种形式。

## 十五、一句话总结这题

这题本质是：

> **利用两个升序数组的单调性，用一个指针从左往右、一个指针从右往左，让“和”朝目标值逼近。**

所以它不是“枚举题”，而是标准的**双指针模板题**。
