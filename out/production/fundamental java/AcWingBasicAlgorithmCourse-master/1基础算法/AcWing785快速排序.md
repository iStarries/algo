## AcWing 785. 快速排序 

`难度：简单`

### 题目描述

给定你一个长度为n的整数数列。

请你使用快速排序对这个数列按照从小到大进行排序。

并将排好序的数列按顺序输出。

**输入格式**

输入共两行，第一行包含整数 n。

第二行包含 n 个整数（所有整数均在$1~10^9$范围内），表示整个数列。

**输出格式**

输出共一行，包含 n 个整数，表示排好序的数列。

**数据范围**

1 ≤ n ≤ 100000

**输入样例**：

```
5
3 1 2 4 5
```

**输出样例**：

```
1 2 3 4 5
```

### Solution

```java 
import java.util.*;
import java.io.*;

public class Main{
    public static void main(String[] args) throws IOException{
        BufferedReader in = new BufferedReader(new InputStreamReader(System.in));
        int n = Integer.parseInt(in.readLine());
        int[] q = new int[n];
        String[] s = in.readLine().split(" ");
        for(int i = 0; i < n; i++){
            q[i] = Integer.parseInt(s[i]);
        }
        quickSort(q, 0, n - 1);
        for(int i = 0; i < n; i++){
            System.out.print(q[i] + " ");
        }
        in.close();
    }
    
    public static void quickSort(int[] q, int l, int r){
        if(l >= r) return; 
        int i = l - 1, j = r + 1, x = q[l + r >> 1];
        while (i < j){
            do i++; while(q[i] < x);
            do j--; while(q[j] > x);
            if(i < j) {
                int t = q[i];
                q[i] = q[j];
                q[j] = t;
            }
        }
        quickSort(q, l, j);
        quickSort(q, j + 1, r);
    }
}
```
这道题是**快速排序模板题**。你的目标不只是把它背下来，而是先弄明白：

1. 快速排序到底在干什么
2. 代码里的 `i`、`j`、`x` 为什么这么写
3. 为什么递归区间是 `l ~ j` 和 `j + 1 ~ r`
4. 以后遇到同类题，怎么自己写出来

这题题目和题解就是标准的快速排序实现。

# 一、先理解：快速排序到底是什么

快速排序的核心思想可以概括成一句话：

**先选一个数作为“分界点”，把数组分成左右两部分：左边都不大于它，右边都不小于它；然后分别继续排左右两边。**

也就是：

* 找一个基准数 `x`
* 让数组重新分布：

  * 左边放小的数
  * 右边放大的数
* 然后对左边再做同样的事
* 对右边再做同样的事

这叫做 **分治**。

# 五、核心函数逐行讲解

## 1）函数定义

```java
public static void quickSort(int[] q, int l, int r)
```

含义：

* `q`：要排序的数组
* `l`：当前处理区间的左端点
* `r`：当前处理区间的右端点

也就是说，这个函数不是每次都排整个数组，而是排某一段。

---

## 2）递归终止条件

```java
if(l >= r) return;
```

这句非常重要。

为什么？

因为当区间里：

* 只有一个数时，天然有序
* 没有数时，也不用排

所以当 `l >= r` 时，直接结束。

---

## 3）初始化三个关键变量

```java
int i = l - 1, j = r + 1, x = q[l + r >> 1];
```

这里最关键。

### `x = q[l + r >> 1]`

这是选基准值，也叫“枢轴”。

`l + r >> 1` 的意思其实是：

```java
(l + r) >> 1
```

也就是 `(l + r) / 2`，取中间下标。

所以：

```java
x = q[(l + r) / 2]
```

表示：**选当前区间中间位置的数作为分界点。**

---

### `i = l - 1, j = r + 1`

这是一种常见写法，配合下面这两句：

```java
do i++; while(q[i] < x);
do j--; while(q[j] > x);
```

作用是让 `i` 从左往右找“应该放右边”的数，`j` 从右往左找“应该放左边”的数。

为什么初值不是 `i = l, j = r`？

因为后面用了 `do...while`，会**先移动一次再判断**。
所以把它们放在区间外一格，正好移动后落到区间边界。

这个写法你现在先记住即可，后面看流程就懂了。

---

# 六、划分过程是整个快速排序的灵魂

```java
while (i < j){
    do i++; while(q[i] < x);
    do j--; while(q[j] > x);
    if(i < j) {
        int t = q[i];
        q[i] = q[j];
        q[j] = t;
    }
}
```

这段代码的目标是：

**把小于 `x` 的数尽量放左边，把大于 `x` 的数尽量放右边。**

---

## 先看第一句

```java
do i++; while(q[i] < x);
```

含义：

* `i` 一直向右走
* 直到找到一个 **大于等于 `x`** 的数，停下来

为什么？

因为左边应该放“小的数”。
如果某个数已经 `< x`，那它待在左边没问题，继续找。
一旦发现 `>= x`，说明它可能放错边了，先停下。

---

## 再看第二句

```java
do j--; while(q[j] > x);
```

含义：

* `j` 一直向左走
* 直到找到一个 **小于等于 `x`** 的数，停下来

因为右边应该放“大的数”。
如果某个数已经 `> x`，它在右边没问题。
一旦发现 `<= x`，说明它可能应该去左边，先停下。

---

## 然后交换

```java
if(i < j) {
    int t = q[i];
    q[i] = q[j];
    q[j] = t;
}
```

这说明：

* 左边找到一个“不该在左边”的数
* 右边找到一个“不该在右边”的数
* 交换它们

这样一步步把数组调整成左右两部分。

---

# 七、拿例子手动模拟一遍

数组：

```text
3 1 2 4 5
```

区间：`l = 0, r = 4`

基准：

```java
x = q[(0 + 4) / 2] = q[2] = 2
```

初始：

```text
i = -1, j = 5
```

---

## 第一轮

### 左边找

`i++` 后 `i = 0`

* `q[0] = 3`
* `3 < 2` 不成立

所以 `i` 停在 0。

### 右边找

`j--` 后 `j = 4`

* `q[4] = 5 > 2`，继续

`j = 3`

* `q[3] = 4 > 2`，继续

`j = 2`

* `q[2] = 2 > 2` 不成立

所以 `j` 停在 2。

此时：

```text
i = 0, j = 2
```

满足 `i < j`，交换 `q[0]` 和 `q[2]`

数组变成：

```text
2 1 3 4 5
```

---

## 第二轮

### 左边找

从 `i = 0` 开始

`i = 1`

* `q[1] = 1 < 2`，继续

`i = 2`

* `q[2] = 3 < 2` 不成立

停在 2。

### 右边找

从 `j = 2` 开始

`j = 1`

* `q[1] = 1 > 2` 不成立

停在 1。

此时：

```text
i = 2, j = 1
```

不满足 `i < j`，循环结束。

---

## 循环结束后说明什么？

此时数组已经被分成两部分：

* `l ~ j`：左半部分
* `j + 1 ~ r`：右半部分

也就是：

* `0 ~ 1`：`2 1`
* `2 ~ 4`：`3 4 5`

注意，这里左半部分不一定完全有序，只是相对于基准完成了划分，所以还要递归处理。

---

# 八、为什么递归写成这样

```java
quickSort(q, l, j);
quickSort(q, j + 1, r);
```

这是很多初学者最容易懵的地方。

循环结束时，`j` 是左半部分的最后一个位置。
所以区间应该分成：

* 左边：`l ~ j`
* 右边：`j + 1 ~ r`

这和这份代码的划分方式是一一对应的。

注意：
**不同写法的快排，递归边界可能不同。**
不能死记别的模板。必须和当前这份“划分逻辑”配套理解。

---

# 九、这份代码为什么是对的

你可以把它理解成下面这个不变目标：

在 `while(i < j)` 的每一轮中，都在努力做到：

* 左边区域尽量都是 `<= x` 的数
* 右边区域尽量都是 `>= x` 的数

最后当 `i >= j` 时，整个区间就被成功分成了两块。
再分别递归排序左右两块，整个数组就有序了。

# 十、这题你应该掌握的重点

对于这题，不是只会抄代码，而是至少要掌握下面几点。

## 1. 快速排序是分治

先划分，再递归处理子问题。

## 2. 基准值 `x` 不一定要放到最终位置

这一点和有些教材讲法不同。
这份模板里，`x` 只是“拿来比较的标准”，并不是说必须把这个元素固定在某个下标上。

## 3. `i` 和 `j` 的职责不同

* `i` 从左往右找 `>= x` 的数
* `j` 从右往左找 `<= x` 的数

## 4. 交换是在纠正“站错边”的元素

左边发现大的，右边发现小的，就交换。

## 5. 递归边界要和划分写法匹配

这里是：

```java
quickSort(q, l, j);
quickSort(q, j + 1, r);
```

# 七、第六步：补上递归

这一步最容易写错。

这份模板循环结束后，递归区间是：

```java

quickSort(q, j + 1, r);
```

完整版本就是：

```java
public static void quickSort(int[] q, int l, int r){
    if(l >= r) return;

    int i = l - 1, j = r + 1;
    int x = q[l + r >> 1];

    while(i < j){
        do i++; while(q[i] < x);
        do j--; while(q[j] > x);
        if(i < j){
            int t = q[i];
            q[i] = q[j];
            q[j] = t;
        }
    }

    quickSort(q, l, j);
    quickSort(q, j + 1, r);
}
```

这就是你要能自己写出来的主模板。

## 1. 终止条件别写错

要写：

```java
if(l >= r) return;
```

不是别的。

---

## 2. `i` 和 `j` 初值别写成 `l` 和 `r`

这里必须是：

```java
int i = l - 1, j = r + 1;
```

因为你后面用的是 `do i++`、`do j--`。

---

## 3. 左右找的条件别反

左边这句是：

```java
do i++; while(q[i] < x);
```

右边这句是：

```java
do j--; while(q[j] > x);
```

一个是 `< x`，一个是 `> x`。
不要两个都写成一样。

---

## 4. 递归边界别写错

这份模板必须是：

```java
quickSort(q, l, j);
quickSort(q, j + 1, r);
```

不是 `i`，也不是 `j - 1`。

因为这是和当前划分方式配套的。

---

# 十、你可以这样理解整段模板

你每次写这段时，脑子里都默念：

* 先选一个基准 `x`
* `i` 从左找大的
* `j` 从右找小的
* 找到就交换
* 最后分成左右两段
* 继续递归

只要这个逻辑不断，你就不容易把代码写散。

# 十一、给你一版“适合背诵”的简洁模板

这是适合你现阶段直接记的版本：

```java
public static void quickSort(int[] q, int l, int r){
    if(l >= r) return;

    int i = l - 1, j = r + 1;
    int x = q[l + r >> 1];

    while(i < j){
        do i++; while(q[i] < x);
        do j--; while(q[j] > x);
        if(i < j){
            int t = q[i];
            q[i] = q[j];
            q[j] = t;
        }
    }

    quickSort(q, l, j);
    quickSort(q, j + 1, r);
}
```

# 十二、再给你一版“带理解提示”的模板

这个版本更适合学习，不是比赛时背的版本：

```java
public static void quickSort(int[] q, int l, int r){
    // 区间里只有 0 个或 1 个数，不用排序
    if(l >= r) return;

    // i 从左边往右走，j 从右边往左走
    int i = l - 1, j = r + 1;

    // 选中间位置的数作为基准值
    int x = q[l + r >> 1];

    while(i < j){
        // 找左边第一个 >= x 的数
        do i++; while(q[i] < x);

        // 找右边第一个 <= x 的数
        do j--; while(q[j] > x);

        // 两边都找到了站错位置的数，就交换
        if(i < j){
            int t = q[i];
            q[i] = q[j];
            q[j] = t;
        }
    }

    // 递归处理左右两部分
    quickSort(q, l, j);
    quickSort(q, j + 1, r);
}
```

# 十三、你现在该怎么练

你现在最有效的练法不是看，而是自己遮住代码，按下面顺序手写 3 遍：

**第一遍：对照写**
把我上面这版模板照着打一遍。

**第二遍：只看骨架写**
只看这五步：

* 终止
* 选基准
* 划分
* 左递归
* 右递归

然后自己补细节。

**第三遍：完全默写**
不看任何内容，自己写完整函数。

只要你能独立写出这个 `quickSort` 函数，这题就基本过关了。

# 十四、最后给你一个小检查题

你写完后，自己检查这 6 个点：

* 有没有 `if(l >= r) return;`
* 有没有 `i = l - 1, j = r + 1`
* 有没有 `x = q[l + r >> 1]`
* 有没有 `while(i < j)`
* 有没有交换
* 递归是不是 `l ~ j` 和 `j + 1 ~ r`

这 6 个都对，模板通常就没问题。
