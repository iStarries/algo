# 什么是面向切面编程（AOP）

AOP 是 **Aspect Oriented Programming** 的缩写，中文叫**面向切面编程**。

我们知道，面向对象编程（OOP）的核心特点是封装、继承和多态。按照 OOP 的思路，不同功能通常会分散在不同的类和方法中。这样做的好处是结构清晰、便于复用，但也会带来一个问题：**一些通用功能容易在多个类、多个方法中重复出现。**

比如，很多业务方法都需要：

* 记录日志
* 做权限校验
* 开启事务
* 统计执行时间

如果按照传统 OOP 的方式来写，这些代码往往要重复写在很多方法里。虽然它们和业务本身关系不大，但却散落在系统各处，导致代码重复、维护困难。

有人可能会想到：把这些重复代码抽到一个单独的方法里，再在各个业务方法中调用不就行了吗？这样确实能减少一些重复，但业务代码仍然要显式调用这些公共逻辑，类与类之间还是会产生耦合。后续一旦公共逻辑发生变化，相关代码也可能要跟着改。

这时，AOP 的价值就体现出来了。

**AOP 的思想就是：把这些分散在各处的公共功能，从业务代码中抽离出来，在程序运行时，动态地插入到指定类的指定方法、指定位置。**

也就是说，开发者只需要关心核心业务逻辑，而日志、事务、权限校验这些“横切”的公共逻辑，可以统一交给 AOP 来处理。

---

# AOP 里的几个关键概念

在 AOP 中：

* **切面（Aspect）**：指那些被抽取出来的公共逻辑，比如日志、事务、权限校验。
* **切入点（Pointcut）**：指这些公共逻辑要作用到哪些类、哪些方法上。
* **通知（Advice）**：指这些公共逻辑在目标方法的什么时机执行，比如方法执行前、执行后、抛异常后等。

你可以把它理解成：

> AOP 就是在不改动原有业务代码的前提下，给目标方法“额外织入”一些公共功能。

---

# AOP 和 OOP 的关系

AOP 并不是用来替代 OOP 的，而是对 OOP 的一种补充。

* **OOP** 主要解决的是“把系统按职责拆分成不同对象”的问题。
* **AOP** 主要解决的是“把多个对象中重复出现的公共行为统一提取出来”的问题。

可以说：

* OOP 是**纵向**组织业务功能
* AOP 是**横向**抽取公共逻辑

两者结合使用，可以让系统结构更清晰，代码更易维护。

---

# 为什么说 AOP 很重要

有了 AOP 之后，我们就可以把原本散落在各个业务类中的重复代码，统一放到切面里管理。这样做有几个明显好处：

* 减少重复代码
* 降低模块之间的耦合
* 让业务代码更纯粹
* 提高维护性和扩展性

例如，以后想给一批方法统一加日志、统一做权限控制、统一加事务处理，不需要逐个去改业务方法，只要修改切面即可。

---

# 一句话总结

**AOP，就是把系统中与业务无关、但会被多个地方重复使用的公共逻辑抽取出来，并在运行时动态织入到目标方法中的一种编程思想。**
