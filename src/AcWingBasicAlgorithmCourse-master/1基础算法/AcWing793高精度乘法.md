# AcWing 算法基础课 -- 基础算法

## AcWing 793. 高精度乘法 

`难度：简单`

### 题目描述

给定两个正整数A和B，请你计算A * B的值。

**输入格式**

共两行，第一行包含整数A，第二行包含整数B。

**输出格式**

共一行，包含A * B的值。


数据范围

$1≤A的长度≤100000,$
$0≤B≤10000$

```r
输入样例：

2
3

输出样例：

6
```
### Solution

```java
import java.util.*;

class Main{
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        String sa = sc.next();
        int la = sa.length();
        int[] a = new int[la];
        // b 比较小，可以用 int 存
        int b = sc.nextInt();
        for(int i = la - 1, j = 0; i >= 0; i--, j++)    a[j] = sa.charAt(i) - '0';
        Deque<Integer> c = new ArrayDeque<>();
        mul(a, b, c);
        while(!c.isEmpty()) System.out.print(c.pop());
    }
    public static void mul(int[] a, int b, Deque<Integer> c){
        int t = 0;
        for(int i = 0; i < a.length; i++){
            t = t + a[i] * b;
            c.push(t % 10);
            t = t / 10;
        }
        // 因为 t 可能很大，所以也要把 t 加到 c 中
        while(t > 0){
            c.push(t % 10);
            t /= 10;
        }
        // 以防是 0 去除前导 0
        while(c.size() > 1 && c.peek() == 0) c.pop();
    }
}
```
这题是**高精度乘法里的“高精度 × 低精度”模板题**。

注意，它不是两个都很大的数相乘，而是：

* `A` 很大，长度最多 `100000`
* `B` 很小，`0 ≤ B ≤ 10000`

所以这题的核心不是“两个高精度相乘”，而是：

**怎么把“一个超大整数乘一个普通整数”用竖式乘法写出来。**

---

# 一、这题考什么

这题主要考三件事：

**1. 高精度数怎么存**
大整数 `A` 不能用 `int` 或 `long`，所以要拆成每一位存起来。

**2. 竖式乘法怎么模拟**
从个位开始乘，每一位都要带上前面的进位。

**3. 进位怎么处理**
这是这题最重要的点。减法考借位，这题考进位。

---

# 二、为什么这题比高精度减法更简单

因为这里是：

```text
高精度 × 低精度
```

也就是一个大数乘一个普通整数。

例如：

```text
123456 × 7
```

你手算时其实就是：

* 6 × 7
* 5 × 7，再加上前一位进位
* 4 × 7，再加上进位
* ……

所以只需要**一层循环**。

如果是“高精度 × 高精度”，那就要更复杂，一般要两层循环。

所以这题本质上是高精度里的入门模板。

---

# 三、整体思路

这题的流程非常固定：

### 第一步：把大整数倒着存

例如：

```text
A = 12345
```

存成：

```text
a = [5, 4, 3, 2, 1]
```

原因还是一样：

**手算乘法是从个位开始的。**

这样 `a[0]` 就是个位，循环从 `0` 开始最自然。

---

### 第二步：从低位到高位依次乘 `B`

设 `t` 表示当前进位。

每一位做：

```text
t = t + a[i] * B
当前位结果 = t % 10
新的进位 = t / 10
```

这个公式你一定要记住。

它就是这题的核心模板。

---

### 第三步：最后把剩余进位处理掉

循环结束后，`t` 可能还不是 `0`。

比如：

```text
999 × 9
```

最后会多出很多高位，所以要继续把 `t` 拆开加入结果。

---

### 第四步：去前导零

因为如果结果是 `0`，可能会出现多余零，所以最后做一次清理。

---

# 四、最关键的问题：为什么这样写

题解核心代码是这个意思：

```java
int t = 0;
for (int i = 0; i < a.length; i++) {
    t = t + a[i] * b;
    c.push(t % 10);
    t = t / 10;
}
while (t > 0) {
    c.push(t % 10);
    t /= 10;
}
```

下面把它彻底讲清楚。

---

# 五、`t` 到底是什么

这里的 `t` 表示：

**当前这一位计算后，暂时还没处理完的值。**

你可以把它理解成：

```text
本位乘积 + 上一位传来的进位
```

比如算：

```text
123 × 4
```

倒序后：

```text
a = [3, 2, 1]
```

初始：

```text
t = 0
```

---

## 第 0 位：3 × 4

```text
t = 0 + 3 * 4 = 12
当前位写下 2
进位变成 1
```

也就是：

```java
c.push(12 % 10);   // 2
t = 12 / 10;       // 1
```

---

## 第 1 位：2 × 4

这时候别忘了还有进位 `1`：

```text
t = 1 + 2 * 4 = 9
当前位写下 9
进位变成 0
```

---

## 第 2 位：1 × 4

```text
t = 0 + 1 * 4 = 4
当前位写下 4
进位变成 0
```

结果就是：

```text
492
```

这就是竖式乘法的程序版。

我们直接推：

```text
1234 × 6
```

倒序存：

```text
a = [4, 3, 2, 1]
b = 6
```

初始：

```text
t = 0
```

## 第 0 位

```text
t = 0 + 4 * 6 = 24
当前位 = 24 % 10 = 4
进位 = 24 / 10 = 2
```

---

## 第 1 位

```text
t = 2 + 3 * 6 = 20
当前位 = 0
进位 = 2
```

---

## 第 2 位

```text
t = 2 + 2 * 6 = 14
当前位 = 4
进位 = 1
```

---

## 第 3 位

```text
t = 1 + 1 * 6 = 7
当前位 = 7
进位 = 0
```

得到的低位到高位是：

```text
[4, 0, 4, 7]
```

翻过来就是：

```text
7404
```

正确。

# 七、为什么结果要用栈 / Deque

因为你是从低位开始算的：

* 先算个位
* 再算十位
* 再算百位

但输出要从高位到低位。

所以题解用了 `Deque`，每次 `push`，最后再 `pop` 输出。

这样能把顺序翻过来。

当然，你也可以：

* 先把结果放数组 / `List`
* 最后倒序输出

本质一样。

---

# 八、为什么最后还要处理 `while (t > 0)`

因为最后可能还有没写完的高位进位。

比如：

```text
999 × 9
```

手算：

```text
9 × 9 = 81，写1进8
9 × 9 + 8 = 89，写9进8
9 × 9 + 8 = 89，写9进8
最后还剩一个 8
```

所以结果是：

```text
8991
```

如果你不写：

```java
while (t > 0)
```

最后那个最高位 `8` 就丢了。

所以这一步绝对不能少。

---

# 九、为什么还要去前导零

主要是为了处理这种情况：

```text
A = 0
B = 1234
```

或者：

```text
A = 12345
B = 0
```

结果应该是：

```text
0
```

而不是：

```text
0000...
```

所以最后要保留一个零，删掉多余前导零。

题解里这句：

```java
while(c.size() > 1 && c.peek() == 0) c.pop();
```

意思就是：

* 只要结果长度大于 `1`
* 并且最高位是 `0`
* 就删掉

但至少保留最后一个 `0`

# 十、这题和高精度减法的区别

你刚学过减法，这里可以顺便对比一下：

## 高精度减法

核心是：

```text
当前位 = 被减数 - 借位 - 减数
```

关注的是：

**借位**

---

## 高精度乘法

核心是：

```text
当前位 = 当前乘积 + 进位
```

关注的是：

**进位**

---

所以你可以这样记：

* 加法：进位
* 减法：借位
* 乘法：乘积带进位
* 除法：余数往下传

这四个高精度模板是一套东西。

---

# 十一、这题最容易错的地方

## 1. 忘记倒序存

不倒序的话，你会从高位开始乘，处理起来很别扭。

## 2. 忘记把进位加进去

很多人会写成：

```java
a[i] * b
```

却忘了加上前面的 `t`。

正确是：

```java
t = t + a[i] * b;
```

---

## 3. 循环结束后忘记处理剩余进位

这会导致最高位缺失。

---

## 4. `B` 看成也是高精度

这题不是两个大数相乘，`B` 很小，直接用 `int` 存就行。

这是题目故意降低难度的点。

---

# 十二、你应该记住的模板

这题你最好直接记住这个逻辑：

```text
1. 字符串读入大整数 A
2. 倒序存入数组
3. 用 int 读入小整数 B
4. 从低位到高位枚举 A 的每一位
5. t += a[i] * B
6. 当前位保存 t % 10
7. t /= 10
8. 最后继续处理剩余进位
9. 去前导零
10. 输出结果
```

---

# 四、如果两个数都很大，怎么做

如果两个数都很大，就不能再用：

```text
枚举 A 的每一位，乘一个普通整数 B
```

因为 `B` 也拆成很多位了。

这时候要用真正的竖式乘法：

## 核心思想

假设：

```text
A = 123
B = 45
```

手算其实是：

```text
123 × 5
123 × 4   （但这是十位，要左移一位）
```

程序里就是：

* `a[i]` 乘 `b[j]`
* 对结果的第 `i + j` 位产生贡献

所以核心式子是：

```text
c[i + j] += a[i] * b[j]
```

然后统一处理进位。

---

# 五、为什么是 `i + j`

因为：

* `a[i]` 表示第 `i` 位
* `b[j]` 表示第 `j` 位

它们相乘后，落在结果的第 `i + j` 位。

例如：

* 个位 × 个位 -> 个位
* 个位 × 十位 -> 十位
* 百位 × 十位 -> 千位

所以就是 `i + j`。

这是高精度乘高精度最核心的一点。

---

# 六、Java 代码：两个数都很大

下面给你一个标准 Java 模板，适合你现在学习使用。

```java
import java.util.*;
import java.io.*;

public class Main {
    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        String A = br.readLine().trim();
        String B = br.readLine().trim();

        int[] a = new int[A.length()];
        int[] b = new int[B.length()];

        // 倒序存储，a[0] / b[0] 表示个位
        for (int i = A.length() - 1, j = 0; i >= 0; i--, j++) {
            a[j] = A.charAt(i) - '0';
        }
        for (int i = B.length() - 1, j = 0; i >= 0; i--, j++) {
            b[j] = B.charAt(i) - '0';
        }

        int[] c = multiply(a, b);

        // 找到最高位，去前导 0
        int k = c.length - 1;
        while (k > 0 && c[k] == 0) k--;

        StringBuilder sb = new StringBuilder();
        for (int i = k; i >= 0; i--) {
            sb.append(c[i]);
        }

        System.out.println(sb.toString());
    }

    public static int[] multiply(int[] a, int[] b) {
        // 最多 a.length + b.length 位
        int[] c = new int[a.length + b.length];

        // 先把每一位乘积累加到对应位置
        for (int i = 0; i < a.length; i++) {
            for (int j = 0; j < b.length; j++) {
                c[i + j] += a[i] * b[j];
            }
        }

        // 统一处理进位
        for (int i = 0; i < c.length - 1; i++) {
            c[i + 1] += c[i] / 10;
            c[i] %= 10;
        }

        return c;
    }
}
```

---

# 七、这段代码怎么理解

## 第一步：倒序存

例如：

```text
A = "123"
B = "45"
```

变成：

```text
a = [3, 2, 1]
b = [5, 4]
```

---

## 第二步：每一位相乘

双层循环：

```java
for (int i = 0; i < a.length; i++) {
    for (int j = 0; j < b.length; j++) {
        c[i + j] += a[i] * b[j];
    }
}
```

意思是：

* `a[i]` 和 `b[j]` 的乘积
* 加到结果第 `i+j` 位

这里先**只累计，不急着进位**。

---

## 第三步：统一进位

例如某一位可能变成了：

```text
c[i] = 27
```

那就要：

* 当前位留下 `7`
* 往高位进 `2`

所以写成：

```java
c[i + 1] += c[i] / 10;
c[i] %= 10;
```

---

# 九、这种写法和“高精度 × 低精度”有什么区别

## 高精度 × 低精度

核心是：

```text
t += a[i] * b
```

只有一层循环，因为 `b` 只是一个普通整数。

---

## 高精度 × 高精度

核心是：

```text
c[i + j] += a[i] * b[j]
```

要两层循环，因为两个数的每一位都要互相乘。

# 十、你以后怎么避免再看错题

以后先看题，不要先看“标题叫高精度乘法”。

而是按这个顺序：

### 第一步

看每个输入的数据范围。

### 第二步

判断每个数应该用什么类型存：

* `int`
* `long`
* `String + 数组`

### 第三步

再确定模板。

你可以这样快速记：

* 两个都是大数：高精度加减 / 高精度乘高精度
* 一个大一个小：高精度乘低精度 / 高精度除低精度

# 十一、最简结论

你没注意到，是因为你只看到了“高精度乘法”，没有先检查：

**哪个数真的需要高精度，哪个数只是普通整数。**

而判断方法只有一个：

**看题目数据范围。**

如果两个数都很大，就用这个核心思路：

```text
c[i + j] += a[i] * b[j]
再统一处理进位
```

如果你愿意，我下一条可以继续给你讲：

**这份“高精度乘高精度”代码每一行在干什么。**





```java
import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
class Main{
    public static void main(String[] args) throws IOException {
        BufferedReader bufferedReader = new BufferedReader(new InputStreamReader(System.in));
        String s1 = bufferedReader.readLine();
        String s2 = bufferedReader.readLine();
        mul(s1, s2);
    }
    public static void mul(String s1, String s2){
        int[] ints1 = new int[s1.length()];
        int int2 = Integer.parseInt(s2);
        int[] res = new int[s1.length()+s2.length()];

        int k = 0;
        for (int i = s1.length() - 1; i >= 0; i--) {
            ints1[k++] = s1.charAt(i) - '0';
        }
        // check if 0
        if(int2 == 0 || ints1[s1.length() - 1] == 0){
            System.out.println(0);
            return;
        }
        for (int i = 0; i < s1.length(); i++) {
            res[i] = ints1[i] * int2;
        }
        for (int i = 0; i < res.length - 1; i++) {
            int t = res[i] % 10;
            res[i + 1] += res[i] / 10;
            res[i] = t;
        }
        int start;
        for (start = res.length - 1; start >= 0; start--) {
            if(res[start] != 0) break;
        }
        while (start >= 0) System.out.print(res[start--]);
    }
}
```
