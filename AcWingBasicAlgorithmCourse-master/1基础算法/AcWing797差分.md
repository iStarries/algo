# AcWing 算法基础课 -- 基础算法

## AcWing 797. 差分 

`难度：简单`

### 题目描述

输入一个长度为n的整数序列。

接下来输入m个操作，每个操作包含三个整数l, r, c，表示将序列中[l, r]之间的每个数加上c。

请你输出进行完所有操作后的序列。

**输入格式**

第一行包含两个整数n和m。

第二行包含n个整数，表示整数序列。

接下来m行，每行包含三个整数l，r，c，表示一个操作。

**输出格式**

共一行，包含n个整数，表示最终序列。


```r
数据范围

1≤n,m≤100000,
1≤l≤r≤n,
−1000≤c≤1000,
−1000≤整数序列中元素的值≤1000

输入样例：

6 3
1 2 2 1 2 1
1 3 1
3 5 1
1 6 1

输出样例：

3 4 5 3 4 2
```

### Solution

```java
import java.util.*;
import java.io.*;

public class Main{
    public static void main(String[] args) throws IOException{
        BufferedReader in = new BufferedReader(new InputStreamReader(System.in));
        String[] s = in.readLine().split(" ");
        // 整数序列 n 个整数
        int n = Integer.parseInt(s[0]);
        // m 个插入操作
        // 插入操作对于数组的索引是从 1 开始的
        // 所以建立差分数组也是从从 1 开始，比较不容易出错
        int m = Integer.parseInt(s[1]);
        s = in.readLine().split(" ");
        int[] a = new int[n + 10];
        for(int i = 1; i <= n; i++){
            int c = Integer.parseInt(s[i - 1]);
            // 插入 c
            insert(a, i, i, c);
        }
        while(m > 0){
            m--;
            s = in.readLine().split(" ");
            int l = Integer.parseInt(s[0]);
            int r = Integer.parseInt(s[1]);
            int c = Integer.parseInt(s[2]);
            insert(a, l, r, c);
        }
        for(int i = 1; i <= n; i++){
            a[i] += a[i - 1];
            System.out.print(a[i] + " ");
        }
        
    }
    
    public static void insert(int[] a, int l, int r, int c){
        a[l] += c;
        a[r + 1] -= c;
    }
    
}
```
这题是基础算法里的“差分”模板题，属于“区间加、最后统一还原”这一类。学习范围也正好在你给的第一讲“前缀和与差分”里。

一、题型与核心考点
这题属于什么类型

数组区间修改

差分数组

前缀和还原原数组

核心考点

这题不是让你每次操作都真的去改 [l, r] 里的每个数，而是用一个辅助数组，把“区间整体加 c”变成两个端点的修改，最后再一次性还原答案。

二、朴素做法 vs 优化做法
1. 朴素做法

每次读到操作 (l, r, c)，就执行：

for (int i = l; i <= r; i++) a[i] += c;

这样每次最坏要改 n 个数，一共 m 次操作：

时间复杂度：O(nm)

这里 n,m ≤ 100000，最坏会到 10^10 级别，肯定超时。

2. 优化做法：差分

差分数组 b 的定义是：

b[1] = a[1]

b[i] = a[i] - a[i - 1]

反过来，

a[i] = b[1] + b[2] + ... + b[i]

也就是说：原数组是差分数组的前缀和。

三、这题的关键思想

如果要给原数组的区间 [l, r] 都加上 c，那么在差分数组里只需要：

b[l] += c

b[r + 1] -= c

为什么？

因为做完前缀和之后：

从 l 开始，后面的值都会多出一个 c

到了 r + 1，再减回去，影响就截止了

所以区间加法被压缩成了两个位置的修改。

# 我没看到差分数组是怎么用的:差分数组不是“每个位置的值”，而是,当前位置相对于前一个位置，变化了多少

你这句话说明了一个很关键的问题：**你看懂了代码在跑，但还没看懂“差分数组到底在中间起了什么作用”**。

那这次我不直接讲整题，我只讲一件事：

> **差分数组是怎么“代替原数组做区间修改”的。**

---

## 一、先别管这题，先看一个最小例子

原数组：

```text
a = [1, 2, 2, 1, 2, 1]
```

对应的差分数组 `b` 是：

```text
b[1] = a[1]
b[2] = a[2] - a[1]
b[3] = a[3] - a[2]
...
```

算出来：

```text
b = [1, 1, 0, -1, 1, -1]
```

因为：

* `1`
* `2 - 1 = 1`
* `2 - 2 = 0`
* `1 - 2 = -1`
* `2 - 1 = 1`
* `1 - 2 = -1`

---

## 二、差分数组到底表示什么

差分数组不是“每个位置的值”，而是：

> **当前位置相对于前一个位置，变化了多少**

比如这个差分数组：

```text
b = [1, 1, 0, -1, 1, -1]
```

它的意思是：

* 第一个数是 1
* 第二个数比前一个大 1，所以变成 2
* 第三个数比前一个大 0，所以还是 2
* 第四个数比前一个小 1，所以变成 1
* 第五个数比前一个大 1，所以变成 2
* 第六个数比前一个小 1，所以变成 1

于是恢复出原数组：

```text
a = [1, 2, 2, 1, 2, 1]
```

所以你要记住：

> **原数组 = 差分数组的前缀和**

也就是：

```java
a[i] = b[1] + b[2] + ... + b[i]
```

---

## 三、差分数组为什么能做区间加法

现在假设我要把原数组的 `[2, 4]` 都加 3。

原数组原来是：

```text
[1, 2, 2, 1, 2, 1]
```

改完应该变成：

```text
[1, 5, 5, 4, 2, 1]
```

你可能会想：那不是得把 2、3、4 位置都改掉吗？

朴素做法确实这样。

但差分的想法是：

> 我不直接改这一整段，我只记录“从哪开始多了 3”和“从哪结束不再多 3”。

---

## 四、只改两个位置，为什么就够了

对区间 `[2, 4] + 3`：

在差分数组里只做：

```text
b[2] += 3
b[5] -= 3
```

为什么是这样？

因为：

* 从第 `2` 位开始，后面的前缀和都应该多 3
* 但只想让它影响到第 `4` 位，所以到第 `5` 位要把这 3 抵消掉

---

## 五、你亲眼看一遍变化过程

原来的差分数组：

```text
b = [1, 1, 0, -1, 1, -1]
```

现在做 `[2, 4] + 3`：

* `b[2] += 3`
* `b[5] -= 3`

得到：

```text
b = [1, 4, 0, -1, -2, -1]
```

然后把它前缀和还原：

* 第 1 位：`1`
* 第 2 位：`1 + 4 = 5`
* 第 3 位：`5 + 0 = 5`
* 第 4 位：`5 + (-1) = 4`
* 第 5 位：`4 + (-2) = 2`
* 第 6 位：`2 + (-1) = 1`

得到：

```text
[1, 5, 5, 4, 2, 1]
```

正好就是我们想要的结果。

---

## 六、所以“差分数组怎么用”的本质就一句话

### 原数组里做区间加：

```text
给 a[l..r] 全部加 c
```

### 等价于在差分数组里做：

```text
b[l] += c
b[r + 1] -= c
```

### 最后再通过前缀和恢复原数组

```java
for (int i = 1; i <= n; i++) {
    b[i] += b[i - 1];
}
```

恢复后，`b[i]` 本身就变成最终答案了。

## 七、回到你那份代码里，它到底哪里在“用差分数组”

你代码里这个数组：

```java
int[] a = new int[n + 10];
```

虽然名字叫 `a`，但它**前面其实不是原数组**，它是**差分数组**。

这一点最容易看晕。

---

### 1. 这句是在构造差分数组

```java
insert(a, i, i, c);
```

这里不是在改原数组，而是在说：

> 原数组第 `i` 个位置初始值是 `c`

把它转成差分形式存进 `a` 里。

因为 `insert(a, i, i, c)` 会执行：

```java
a[i] += c;
a[i + 1] -= c;
```

这就是把“单点加 c”记到差分数组里。

---

### 2. 这句是在做区间修改

```java
insert(a, l, r, c);
```

执行的是：

```java
a[l] += c;
a[r + 1] -= c;
```

这一步仍然是在**修改差分数组**，不是直接修改原数组。

也就是说，所有操作都只是记在差分数组里。

---

### 3. 这句才是在还原最终原数组

```java
a[i] += a[i - 1];
```

到了这里，前缀和一做，`a` 才从“差分数组”变回“最终原数组”。

所以你的代码流程其实是：

### 第一步

`a[]` 当作差分数组来存信息

### 第二步

所有区间操作都改 `a[]` 的两个端点

### 第三步

最后做前缀和，把 `a[]` 还原成真正答案

---

## 八、你之所以没看出来，是因为变量名把你骗了

代码里写的是：

```java
int[] a = new int[n + 10];
```

但它其实经历了两个身份：

### 前半段

它是 **差分数组**

### 后半段

做完前缀和之后，它才变成 **最终原数组**

所以更清楚的写法应该是：

```java
int[] diff = new int[n + 10];
```

这样你一眼就知道：

* `insert(diff, l, r, c)` 是在改差分
* `diff[i] += diff[i - 1]` 是在还原答案

---

## 九、我用你这题的代码逻辑，重写成“更容易看懂”的版本

```java
static void insert(int[] diff, int l, int r, int c) {
    diff[l] += c;
    diff[r + 1] -= c;
}
```

这句表示：

> 在差分数组 diff 上记录一次“区间 [l, r] 加 c”

然后：

```java
for (int i = 1; i <= n; i++) {
    int x = ...;   // 原数组第 i 个数
    insert(diff, i, i, x);
}
```

这一步是把原数组先转成差分数组。

接着：

```java
insert(diff, l, r, c);
```

这一步是记录每次区间修改。

最后：

```java
for (int i = 1; i <= n; i++) {
    diff[i] += diff[i - 1];
}
```

这一步是把差分数组还原成最终数组。

# 这种差分能用在什么题型里

这种一维差分，最适合的题型其实很集中。你可以先记一个判断标准：

> **有很多次“区间一起加/减”，并且最后统一求结果**
> 这时候就很可能能用差分。

---

## 一、最典型的题型

### 1. 区间加法题

例如：

* 给长度为 `n` 的数组
* 进行 `m` 次操作
* 每次把 `[l, r]` 都加上 `c`
* 最后输出整个数组

这就是差分的标准题型，也是你现在这道题的类型。

---

### 2. 区间减法题

本质和区间加法一样，因为“减去 `c`”就是“加上 `-c`”。

例如：

* `[l, r]` 每个数都减 5
* 那就在差分里做
  `diff[l] -= 5`
  `diff[r + 1] += 5`

所以只要是**区间整体增减**，都能用。

---

### 3. 多次修改，最后问每个位置的最终值

这种题常见表述是：

* 一开始有一个数组
* 后面有很多操作
* 最后问第 `i` 位是多少，或者输出整个数组

这类题本质上还是“最后统一结算”，非常适合差分。

---

## 二、稍微变形一点的题型

### 1. 给若干区间，问每个位置被覆盖了多少次

比如：

* 有很多区间 `[l, r]`
* 每个区间表示“这一段被覆盖一次”
* 最后问每个点被覆盖了几次

这种题也能用差分。

因为每出现一个区间 `[l, r]`，你就做：

```java
diff[l] += 1;
diff[r + 1] -= 1;
```

最后前缀和后，`diff[i]` 就表示：

> 第 `i` 个位置被覆盖了多少次

这是差分非常经典的一种用法。

---

### 2. 统计某些区间内是否被影响过

比如：

* 某些操作会影响 `[l, r]`
* 最后判断哪些位置被影响过
* 或者有多少位置至少被影响一次

本质上也是“区间打标记”，所以仍然可以差分。

---

### 3. 航班预订、乘客上下车、人数变化题

这类题常常会包装成故事题。

比如：

* 第 `l` 天到第 `r` 天每天增加 `c` 个订单
* 某辆车在第 `l` 站上来 `c` 人，在第 `r+1` 站以后不再算这些人
* 某个时间段内流量整体增加

这些题看起来不是数组题，但本质还是：

> **某一段连续区间都加同一个值**

所以都能转成差分。

---

## 三、二维差分对应什么题型

如果题目从数组变成矩阵，就会变成**二维差分**。

### 典型题型

* 给一个 `n × m` 的矩阵
* 多次把某个子矩阵 `[x1, y1] ~ [x2, y2]` 全部加上 `c`
* 最后输出整个矩阵

这就是一维差分的升级版。

你现在先把一维差分吃透，之后二维差分会更容易理解。

---

## 四、哪些题一般不能只用这种差分

这个也很重要。不是看到区间就一定差分。

### 1. 每次修改后立刻查询

比如：

* 修改一次就问一次当前区间和
* 修改一次就问第 `k` 个数是多少

这种题如果是**边改边查**，普通差分通常不够。

因为差分适合的是：

> 先把所有修改记下来，最后统一还原

如果每次都要立即查，往往要考虑：

* 树状数组
* 线段树

---

### 2. 区间乘法、区间最值、复杂操作

差分最擅长的是：

* 区间加同一个数
* 区间减同一个数

但如果变成：

* 区间乘一个数
* 区间取最大值
* 区间赋值成某个数

就不是这个基础差分模板能直接解决的了。

---

### 3. 区间不连续

差分要求影响的是一个**连续区间**。

如果题目说：

* 修改所有奇数位置
* 修改所有满足某条件的位置

这种不是连续段，就不能直接套一维差分。

---

## 五、你做题时怎么识别“该想到差分”

看到下面几个信号，就应该往差分上想：

### 信号 1：操作是对一整段同时加减

例如：

* `[l, r]` 全加 `c`
* `[l, r]` 全减 `c`

这是最强信号。

### 信号 2：操作很多

如果 `m` 很大，暴力改区间会超时。

### 信号 3：最后统一输出

不是每次操作后都查询，而是最后一次性求答案。

### 信号 4：要求每个点最终受了多少影响

这种“累计影响”的味道，也很像差分。

---

## 六、你可以把它记成一个题型模板

### 能用差分的常见表达

* 多次对区间 `[l, r]` 加 `c`
* 多次对区间 `[l, r]` 减 `c`
* 多个区间覆盖，求每个点被覆盖次数
* 多段时间/路段/日期统一增加某个量
* 所有操作结束后求最终状态

---

## 七、一句话总结

**一维差分最适合：连续区间批量修改，最后统一结算。**

你可以把它和前缀和对着记：

* **前缀和**：多次查询区间和
* **差分**：多次修改区间值

---

## 八、给你一个判断口诀

> **区间加减很多次，最后统一看结果 —— 差分。**

你要是愿意，我下一条可以继续给你讲：

**“前缀和和差分到底是什么关系”**
这个一旦想通，你以后题型就更容易分辨了。



```java
import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;

class Main {
    public static void main(String[] args) throws IOException {
        BufferedReader bufferedReader = new BufferedReader(new InputStreamReader(System.in));
        String[] s1 = bufferedReader.readLine().split(" ");
        int n = Integer.parseInt(s1[0]);
        int m = Integer.parseInt(s1[1]);
        int[] arr = new int[n + 1];
        int[] res = new int[n + 1];
        String[] s2 = bufferedReader.readLine().split(" ");
        for (int i = 1; i <= n; i++) {
            arr[i] = Integer.parseInt(s2[i - 1]);
            res[i] = arr[i] - arr[i - 1];
        }
        String[] s;
        while (m > 0) {
            m--;
            s = bufferedReader.readLine().split(" ");
            int l = Integer.parseInt(s[0]);
            int r = Integer.parseInt(s[1]);
            int c = Integer.parseInt(s[2]);
            res[l] += c;
            if(r < n)res[r + 1] -= c;
        }
        int t = 0;
        for (int i = 1; i < res.length; i++) {
            t += res[i];
            System.out.print(t+" ");
        }
    }
}
```
