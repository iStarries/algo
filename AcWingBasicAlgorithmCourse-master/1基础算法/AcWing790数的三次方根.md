# AcWing 算法基础课 -- 基础算法

## AcWing 790. 数的三次方根

`难度：简单`

### 题目描述

给定一个浮点数n，求它的三次方根。

**输入格式**

共一行，包含一个浮点数n。

**输出格式**

共一行，包含一个浮点数，表示问题的解。

注意，结果保留6位小数。

**数据范围**：
```r
−10000≤n≤10000
```
**输入样例**：

```r
1000.00
```

**输出样例**：

```r
10.000000
```

### Solution

```java
import java.io.*;

public class Main{
    public static void main(String[] args) throws IOException{
        final double ACC = 1e-8;
        BufferedReader in = new BufferedReader(new InputStreamReader(System.in));
        double x = Double.parseDouble(in.readLine());
        double l = -100, r = 100;
        while(r - l > ACC){
            double mid  = (l + r) / 2;
            if(mid * mid * mid > x) r = mid;
            else l = mid;
        }
        System.out.printf("%.6f", l);
    }
}
```
这题是**二分答案在实数范围上的模板题**，和你刚学的“整数二分找边界”是同一家族，只不过现在不是在数组下标里二分，而是在**答案值本身**上二分。题目和代码在你上传的文件里。

---

## 一、这题在问什么

给一个浮点数 `n`，求它的三次方根。

比如：

```text
1000 -> 10
因为 10^3 = 1000
```

如果是：

```text
8 -> 2
-8 -> -2
```

所以本质上是求一个数 `y`，使得：

```java
y * y * y == n
```

当然，浮点数通常不可能刚好算到完全相等，所以我们只能求一个**足够接近的值**。

---

## 二、为什么能用二分

因为函数

```java
f(y) = y^3
```

是**单调递增**的：

* `y` 越大，`y^3` 越大
* `y` 越小，`y^3` 越小

既然单调，就可以二分。

这和你前面学的二分本质一样：

* 如果 `mid^3 > n`，说明答案在左边
* 如果 `mid^3 < n`，说明答案在右边

所以我们可以在一个区间里不断缩小范围，最后逼近答案。

---

## 三、为什么区间是 `[-100, 100]`

代码里写的是：

```java
double l = -100, r = 100;
```

因为题目给了：

```text
-10000 <= n <= 10000
```

而：

```text
100^3 = 1000000
(-100)^3 = -1000000
```

显然 `[-100, 100]` 一定已经把答案包住了。

比如 `10000` 的三次方根大约是 `21.5`，也在这个区间里。
所以这个区间够大，安全。

---

## 四、代码逐行讲

### 1. 精度

```java
final double ACC = 1e-8;
```

表示当区间长度小到 `1e-8` 时，就认为已经足够精确，可以停下来了。

因为题目只要求输出 6 位小数，所以 `1e-8` 是够用的。

---

### 2. 读入

```java
BufferedReader in = new BufferedReader(new InputStreamReader(System.in));
double x = Double.parseDouble(in.readLine());
```

读入一个浮点数 `x`。

这里：

* `Double.parseDouble(...)` 是把字符串转成 `double`

---

### 3. 初始化二分区间

```java
double l = -100, r = 100;
```

表示我们认为答案一定在 `[l, r]` 里。

---

### 4. 二分循环

```java
while(r - l > ACC){
    double mid = (l + r) / 2;
    if(mid * mid * mid > x) r = mid;
    else l = mid;
}
```

这是这题最核心的地方。

#### 为什么不是 `l <= r`

因为这里二分的是**实数**，不是整数。

整数二分时，区间能一个一个缩小到相邻下标。
但实数有无限多个，不能写 `l <= r`，否则可能一直循环。

所以实数二分的停止条件通常写成：

```java
while (r - l > 精度)
```

意思是：只要区间还不够小，就继续缩。
区间足够小时，就停。

---

### 5. 判断逻辑

```java
if(mid * mid * mid > x) r = mid;
else l = mid;
```

因为我们找的是三次方根 `y`，满足：

```java
y^3 = x
```

如果当前 `mid^3 > x`，说明 `mid` 偏大，答案在左边，所以：

```java
r = mid;
```

如果当前 `mid^3 <= x`，说明 `mid` 还不大于答案，所以：

```java
l = mid;
```

注意这里不是 `mid + 1`、`mid - 1`，因为这是实数二分，没有“下一个整数”这种概念。

---

### 6. 输出

```java
System.out.printf("%.6f", l);
```

输出 6 位小数。

这里输出 `l` 或 `r` 都可以，因为最后两者已经非常接近了。

---

## 五、为什么最后输出 `l`

这和你前面问的“为什么返回 `l` / `r`”其实有联系。

这段代码维护的是：

* `l` 始终是一个**立方后不超过 `x`** 的值
* `r` 始终是一个**立方后大于 `x`** 的值

也就是：

```text
l^3 <= x
r^3 > x
```

当区间已经很小的时候：

* `l` 很接近答案
* `r` 也很接近答案

所以输出谁都可以。这里只是代码里选了输出 `l`。

## 六、手推一个例子：`x = 8`

初始：

```text
l = -100, r = 100
```

第一次：

```text
mid = 0
0^3 = 0 <= 8
所以 l = 0
```

第二次：

```text
mid = 50
50^3 > 8
所以 r = 50
```

第三次：

```text
mid = 25
25^3 > 8
所以 r = 25
```

继续下去，区间会越来越小，最后逼近：

```text
2.000000
```

## 七、这题和你前面学的整数二分有什么区别

### 相同点

都是利用“单调性”来缩小答案范围。

### 不同点

#### 整数二分

常见写法：

```java
while (l <= r)
```

更新是：

```java
l = mid + 1
r = mid - 1
```

因为整数有明确的相邻位置。

#### 实数二分

常见写法：

```java
while (r - l > eps)
```

更新是：

```java
l = mid
r = mid
```

因为实数没有“+1”或“-1”这种边界跳跃。

---

## 八、这题最该记住的模板

你可以把它记成下面这个通用形式：

```java
double l = 左边界, r = 右边界;
while (r - l > eps) {
    double mid = (l + r) / 2;
    if (check(mid)) r = mid;
    else l = mid;
}
```

关键在于：

### 先想清楚 `check(mid)` 表示什么

这题里可以理解为：

```java
check(mid): mid^3 > x
```

如果成立，说明 `mid` 偏大，往左缩。
否则往右缩。

---

## 九、为什么负数也能处理

因为三次方和平方不一样。

* 平方后永远不小于 0
* 但三次方保留正负号

例如：

```text
(-2)^3 = -8
```

所以即使 `x` 是负数，三次方根仍然存在，而且函数 `y^3` 仍然单调递增。
这就是为什么这题可以直接统一二分，不用分正负讨论。

---

## 十、你现在最该掌握的核心

这题真正要学的是：

### 1. 不是只有“数组”才能二分

只要答案具有单调性，就能二分答案。

### 2. 实数二分和整数二分的停止条件不同

* 整数：看 `l <= r`
* 实数：看区间长度是否足够小

### 3. 实数二分更新时不用 `+1`、`-1`

因为实数没有离散边界。

## 十一、这题属于你目录里的哪一类

这题仍然属于**基础算法里的二分**，只是从“有序数组上的二分”进阶到“实数域上的二分答案”。

## 十二、给你一个最短记忆版

你可以先背这一版：

```java
求实数解：
1. 先找一个一定包含答案的区间 [l, r]
2. 只要 r - l 还很大，就取 mid
3. 根据 mid 是否偏大，缩左边或右边
4. 最后输出 l 或 r
```

## 我还是没有理解核心，到底怎么判断什么题算是二分答案题？
你卡住的点很正常。

因为“二分答案”最难的，不是会不会写代码，而是：

**看到题目时，脑子里能不能立刻意识到：这个“答案”可以拿来猜。**

我直接告诉你一个最实用的判断方法，不绕。

---

# 一、先说最核心的判断标准

一道题是不是二分答案题，关键看这两件事：

## 1. 题目是不是在问一个“值”

比如问：

* 最小是多少
* 最大是多少
* 至少多少
* 至多多少
* 能不能达到某个数值
* 某个实数解是多少

也就是，题目求的不是“位置”，而是一个**答案值**。

---

## 2. 你能不能“猜一个答案 mid”，然后判断它行不行

也就是：

**假设答案是 mid，能不能检查这个 mid 是否可行。**

如果能，就继续看第三步。

---

## 3. 这个“可行/不可行”是不是单调的

也就是会不会出现这种结构：

```text
不行 不行 不行 行 行 行
```

或者：

```text
行 行 行 不行 不行 不行
```

只要是这种“一刀切开”的结构，通常就能二分。

---

# 二、最短判断口诀

你可以先死记这一句：

**求最值，能检验，有单调，就二分答案。**

这是最实用的。

---

# 三、为什么很多人会分不清

因为“二分答案”不是看题目表面有没有“二分”两个字，而是看你能不能把题目改写成：

> 假设答案是 `mid`，问你：这样行不行？

只要你能这样改写，题就开始有二分答案的味道了。

---

# 四、我给你一个非常硬的判断流程

以后你拿到题，按这个顺序问自己：

## 第一步：题目求的是不是最小值 / 最大值 / 临界值

如果是，先警觉。

例如：

* 最大绳子长度
* 最小机器能量
* 最小时间
* 最大平均值
* 最小半径

这些都很像二分答案。

---

## 第二步：我能不能假设答案是 mid

比如：

* 假设绳长是 `mid`
* 假设最少时间是 `mid`
* 假设容量是 `mid`
* 假设跳跃初始能量是 `mid`

如果能假设，继续。

---

## 第三步：假设后，能不能快速判断“行不行”

这一步特别关键。

例如：

* 长度为 `mid` 时，能不能切出至少 `k` 段
* 时间为 `mid` 时，能不能完成任务
* 容量为 `mid` 时，能不能在规定天数内运完
* 初始能量为 `mid` 时，机器人会不会死

如果能写出一个 `check(mid)`，继续。

---

## 第四步：check 的结果是不是单调变化

比如：

* `mid` 越大越容易行
* 或 `mid` 越大越不容易行

只要方向固定，就是单调。

这时基本就可以确定是二分答案题。

---

# 五、你真正该盯住的是“check(mid)”

其实二分答案题的灵魂不是“二分”，而是：

**你能不能设计出 `check(mid)`。**

因为如果 `check(mid)` 都写不出来，就没法二分。

所以以后不要先想“能不能二分”，要先想：

**我猜一个答案之后，怎么验证它。**

---

# 六、举两个对比，你马上就清楚

## 例1：这是二分答案题

有若干绳子，问最长能切成多长，才能至少得到 `k` 段。

你可以猜：

```text
假设每段长度是 mid
```

然后检查：

```text
能不能切出至少 k 段
```

而且：

* `mid` 越小，越容易可行
* `mid` 越大，越难可行

这是单调的。

所以它是二分答案题。

---

## 例2：这通常不是二分答案题

给你一个无序数组，问有没有两个数和等于 `k`。

你没法自然地去猜一个“答案值 mid”然后判断它行不行。
而且“可行性”也没有形成单调边界。

所以这题不是二分答案题。

---

# 七、你可以把题分成两类

## 第一类：找东西

比如：

* 某个数在不在数组里
* 第一次出现的位置
* 最后一次出现的位置

这是普通二分查找。

---

## 第二类：猜答案

比如：

* 最小时间
* 最大长度
* 最小容量
* 最小初始值
* 某个实数解

这是二分答案。

---

# 八、什么题最容易是二分答案题

你看到下面这些说法时，要立刻敏感：

* 最小的……使得……
* 最大的……使得……
* 至少……时，最优值是多少
* 是否能在……条件下完成
* 给定一个值，问能不能做到
* 求单调函数的解

这些都是强信号。

---

# 九、什么叫“单调”，你一定要真正懂

单调不是说数字本身有序，而是说：

**当你猜的答案变大或变小时，`check(mid)` 的真假不会来回反复横跳。**

例如：

```text
mid = 1  行
mid = 2  行
mid = 3  行
mid = 4  不行
mid = 5  不行
```

这叫单调。

但如果是：

```text
mid = 1  行
mid = 2  不行
mid = 3  行
```

这种乱跳，就不能二分答案。

# 十、一个最关键的反应训练

以后见题，先别问：

**“能不能二分？”**

先问：

**“如果我猜答案是 mid，我能检查它吗？”**

这是核心中的核心。

因为二分答案本质就是：

* 猜一个值
* 检查这个值
* 利用单调性缩范围

---

# 十一、你现在可以这样粗暴判断

我给你一个非常适合初学者的版本：

## 满足下面三条，八成就是二分答案题：

### 1. 题目求最值或临界值

不是求具体方案，不是求某个位置。

### 2. 可以写出 `check(mid)`

能判断“答案取 mid 时是否成立”。

### 3. `check(mid)` 是单调的

一边全 true，一边全 false。

---

# 十二、反过来，什么情况通常不是

通常不是二分答案题的情况：

* 题目求的是具体排列、具体路径、具体方案
* 你根本猜不出一个“答案值”
* 猜了也没法检查
* 检查结果不单调

比如很多 DFS、DP、图搜索题，往往不是直接二分答案。

---

# 十三、再压缩成一句人话

你可以这样理解：

**二分答案题，就是“答案虽然不会直接求，但可以靠试出来，而且试出来的结果有明确分界线”。**

2. 我能不能假设答案是 `mid`？
3. 我能不能判断这个 `mid` 是否可行，并且可行性是单调的？

三个“是”，基本就是二分答案。

答案：2,4,5


```java
import java.io.*;
public class Main {
    public static void main(String[] args) throws IOException {
        BufferedReader bufferedReader = new BufferedReader(new InputStreamReader(System.in));
        double num = Double.parseDouble(bufferedReader.readLine());
        find(num);
    }
    private static void find(double num) {
        double acc = 1e-8;
        double l = -100;
        double r = 100;
        double mid = 0;

        while (r - l > acc){
            mid = (l + r) / 2;
            if (mid * mid * mid > num) r = mid;
            else l = mid;
        }
        System.out.println(String.format("%.6f", l));
    }
}
```
