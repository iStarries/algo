## AcWing 143. 最大异或对

`难度：简单`

### 题目描述

在给定的N个整数A1，A2……AN

中选出两个进行xor（异或）运算，得到的结果最大是多少？

**输入格式**

第一行输入一个整数N。

第二行输入N个整数A1～AN。

**输出格式**

输出一个整数表示答案。

**数据范围**

$1≤N≤10^5,$
$0≤Ai<2^31$

```r
输入样例：

3
1 2 3

输出样例：

3
```

### Solution

```java
import java.util.*;
import java.io.*;

class Main{
    static final int N = 100010;
    static final int M = 31 * N;
    static int[][] son = new int[M][2];
    static int[] a = new int[N];
    static int idx = 0;
    public static void insert(int x){
        int p = 0;
        for(int i = 30; i >= 0; i--){
            int t = x >> i & 1;
            if(son[p][t] == 0) son[p][t] = ++idx;
            p = son[p][t];
        }
    }
    public static int query(int x){
        int res = 0;
        int p = 0;
        for(int i = 30; i >= 0; i--){
            int t = x >> i & 1;
            if(son[p][t ^ 1] != 0) {
                p = son[p][t ^ 1];
                res = res * 2 + (t ^ 1);
            }
            else {
                p = son[p][t];
                res = res * 2 + t;
            }
        }
        return res;
    }
    public static void main(String[] args) throws IOException{
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        String[] s = br.readLine().split(" ");
        int n = Integer.parseInt(s[0]);
        s = br.readLine().split(" ");
        for(int i = 0; i < n; i++) a[i] = Integer.parseInt(s[i]);
        int res = 0;
        for(int i = 0; i < n; i++){
            insert(a[i]);
            int t = query(a[i]);
            res = Math.max(res, a[i] ^ t);
        }
        System.out.println(res);
    }
} 
```

## 一、题型与核心考点

这题是一个非常典型的 **“最大异或值”** 问题。

### 题型特征

当你看到下面这些关键词时，就要往这类做法上想：

* 给定很多整数
* 选两个数
* 让它们的 `xor / 异或` 结果最大
* 数据范围很大，`N ≤ 10^5`

### 核心考点

这题的核心不是普通 Trie，而是 **二进制 Trie / 01-Trie**：

* 每个数看成一个二进制串
* 从高位到低位插入 Trie
* 查询某个数时，尽量去走“相反的位”，让异或结果更大

这是第二讲数据结构里 Trie 的一个非常经典的变形应用。

---

## 二、核心思路与整体做法

---

### 1. 先想朴素做法

最直接的想法就是：

* 枚举两个数 `Ai` 和 `Aj`
* 计算 `Ai ^ Aj`
* 取最大值

时间复杂度是：

[
O(N^2)
]

因为有 `10^5` 个数，两两枚举大约是 `10^{10}` 级别，肯定超时。

所以这题的关键就在于：

> **怎么快速找到一个数最适合和谁异或。**

---

### 2. 为什么要看二进制

异或的大小，本质上取决于二进制表示。

比如两个数异或后，越高位出现 `1`，结果就越大。
所以我们在比较两个异或结果大小时，优先看高位。

这就启发我们：

> 对于某个数 `x`，如果想让 `x ^ y` 最大，那么应该从高位开始，尽量让 `x` 当前位和 `y` 当前位不同。

因为：

* `1 ^ 0 = 1`
* `0 ^ 1 = 1`

而异或结果某一位如果是 `1`，这一位贡献就更大。

---

### 3. 01-Trie 是怎么帮我们做这件事的

把所有数都拆成二进制，从最高位到最低位插入 Trie。

Trie 的每一层只会有两个分支：

* `0`
* `1`

所以叫 **01-Trie**。

然后对于某个数 `x` 去查询时：

* 如果当前位是 `0`，那最好找 `1`
* 如果当前位是 `1`，那最好找 `0`

也就是：

* 优先走 `t ^ 1`
* 如果没有，再退而求其次走 `t`

这样就能保证：

> 从最高位开始，尽量让异或结果的每一位都大。

---

### 4. 整体做法

整体流程就是：

1. 依次读入每个数
2. 先把它插入 Trie
3. 再在 Trie 中找一个和它异或值尽量大的数
4. 更新答案

你这份代码里 `query(x)` 返回的不是“最大异或值”，而是：

> **Trie 中与 `x` 匹配出来的那个数本身**

所以最后用：

```java
a[i] ^ t
```

得到实际异或值。

---

## 三、题解代码实现讲解

下面按函数来拆。

---

### 1. Trie 的存储方式

```java
static final int N = 100010;
static final int M = 31 * N;
static int[][] son = new int[M][2];
static int[] a = new int[N];
static int idx = 0;
```

这里用的是 **静态数组模拟 Trie**。

#### 含义

* `son[p][0]`：节点 `p` 的 `0` 儿子
* `son[p][1]`：节点 `p` 的 `1` 儿子
* `idx`：当前已经创建到哪个节点

#### 为什么是 `31 * N`

因为题目里：

[
0 \le Ai < 2^{31}
]

所以一个数最多看 **31 位**，也就是从第 `30` 位到第 `0` 位。

每插入一个数，最多新建 31 个节点，所以总节点数开 `31 * N` 就够了。

---

### 2. 插入操作 `insert`

```java
public static void insert(int x){
    int p = 0;
    for(int i = 30; i >= 0; i--){
        int t = x >> i & 1;
        if(son[p][t] == 0) son[p][t] = ++idx;
        p = son[p][t];
    }
}
```

### 这段在做什么

把整数 `x` 的二进制，从高位到低位插入 Trie。

### 过程

* 从根节点 `p = 0` 开始
* 枚举每一位 `i`
* 取出这一位二进制值 `t`
* 如果当前节点没有这条分支，就新建
* 然后往下走

### 为什么一定从高位到低位

因为我们想比较异或结果大小时，高位优先。
Trie 也必须按这个顺序组织，查询时才能优先贪心处理高位。

---

### 3. 查询操作 `query`

```java
public static int query(int x){
    int res = 0;
    int p = 0;
    for(int i = 30; i >= 0; i--){
        int t = x >> i & 1;
        if(son[p][t ^ 1] != 0) {
            p = son[p][t ^ 1];
            res = res * 2 + (t ^ 1);
        }
        else {
            p = son[p][t];
            res = res * 2 + t;
        }
    }
    return res;
}
```

这是这题最关键的部分。

### 它的目标

在 Trie 里找一个数 `y`，使得 `x ^ y` 尽量大。

### 贪心策略

对于 `x` 当前位 `t`：

* 最优：找 `t ^ 1`
* 次优：只能找 `t`

因为异或希望当前位尽量为 `1`。

---

### 为什么 `res = res * 2 + ...`

这里是在把找到的那个数 `y` 一位一位拼出来。

比如：

* 原来 `res` 表示已经拼好的高位部分
* 现在再加一位，相当于左移一位再加当前位

所以：

```java
res = res * 2 + 当前选择的位
```

最终 `res` 就是 Trie 中找到的那个最佳匹配数。

注意：

> 这里返回的是“匹配到的数”，不是“异或值”。

所以主函数里要再做一次：

```java
a[i] ^ t
```

---

### 4. 主函数

```java
public static void main(String[] args) throws IOException{
    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
    String[] s = br.readLine().split(" ");
    int n = Integer.parseInt(s[0]);
    s = br.readLine().split(" ");
    for(int i = 0; i < n; i++) a[i] = Integer.parseInt(s[i]);
    int res = 0;
    for(int i = 0; i < n; i++){
        insert(a[i]);
        int t = query(a[i]);
        res = Math.max(res, a[i] ^ t);
    }
    System.out.println(res);
}
```

### 主流程

* 读入所有数
* 枚举每个数 `a[i]`
* 插入 Trie
* 查询当前 Trie 里和它异或最优的数
* 更新答案

---

## 四、这份代码为什么是对的

核心正确性来自一句话：

> **异或结果大小比较，本质上是二进制高位优先比较。**

所以对于某个数 `x`：

* 在第 30 位，能让异或结果变成 `1`，就一定优先这么做
* 如果做不到，再考虑 `0`
* 然后继续考虑下一位

这就是一个标准的 **高位优先贪心**。

而 Trie 恰好可以让我们在每一位上快速知道：

* 想走的分支存不存在

因此查询一个数只需要扫 31 位，复杂度是：

[
O(31)=O(1)
]

总复杂度：

* 插入 `N` 次
* 查询 `N` 次
* 每次都是 31 层

所以总复杂度：

[
O(N \times 31)
]

也就是近似 `O(N)`，完全可以通过。

---

## 五、这题为什么能想到这种解法

这一部分最重要，是做题识别能力。

### 1. 看到“异或最大”，先想到二进制

因为异或本来就是按位运算，结果大小也由二进制高位决定。
所以这类题首先就要有反应：

> **不能只看十进制大小，要转成二进制按位考虑。**

---

### 2. 看到“从高位开始贪心选择”，想到 Trie

当你发现：

* 每一位都只可能是 `0/1`
* 想从高位到低位逐位决策
* 还要快速判断“某种前缀是否存在”

这就是 Trie 的经典使用信号。

普通 Trie 存字符串，这里只是把二进制位串当成“字符串”来存。

---

### 3. 朴素做法和优化做法的本质区别

#### 朴素做法

* 枚举“和谁配对”
* 一个一个试

#### Trie 做法

* 不再枚举配对对象
* 改为在二进制前缀树上“按位贪心找最优对象”

也就是说，优化的关键不是数学公式，而是：

> **把“找最佳配对数”这个问题，变成在 Trie 上走一条最优路径。**

---

### 4. 以后遇到什么题也要想到它

下面这些题型都要联想到 01-Trie：

* 求数组中两数最大异或值
* 给一个数，多次询问和集合里哪个数异或最大
* 区间异或、前缀异或后求最大值
* 动态插入数字，实时求最大异或

只要题目核心变成：

> **在一堆数里，快速找一个数，使它和当前数的异或最优**

基本就该往 **01-Trie** 上靠。

# 从高到低存储二进制位要怎么使用?
你卡住的点很正常。
你不是没理解“树”，而是没理解：

> **把一个数拆成从高位到低位的一串 0/1 之后，这棵树到底怎么拿来“找答案”。**

我这次不直接讲代码，先只讲“怎么使用”。

---

# 一、先别管题，先看“存”的动作到底是什么

假设我们只看 **4 位二进制**，这样好理解。

有三个数：

* 3 = `0011`
* 5 = `0101`
* 6 = `0110`

把它们从高位到低位插入 Trie，就是把每个数当成一条路径：

* `0011`
* `0101`
* `0110`

插进去以后，大概长这样：

```text
(root)
 └── 0
      ├── 0
      │    └── 1
      │         └── 1      ← 0011 = 3
      └── 1
           ├── 0
           │    └── 1      ← 0101 = 5
           └── 1
                └── 0      ← 0110 = 6
```

你可以先这样理解：

* 每往下一层，就表示“确定了一位”
* 一条从根走到底的路径，就是一个完整的二进制数

所以“从高到低存”这句话，本质就是：

> **把一个数的每一位按顺序挂到树上。**

---

# 二、存好了以后，到底怎么“用”？

这才是关键。

这题要做的是：

> 对于一个数 `x`，去树里找另一个数 `y`，让 `x ^ y` 最大。

所以 Trie 的用途不是“把数存起来好看”，而是：

> **让我们能按位贪心地找最优匹配数。**

---

# 三、为什么一定要从高位开始用？

因为异或结果比大小时，**高位最重要**。

例如：

```text
1000 > 0111
```

虽然下面很多位不同，但只要最高位先大，整体就大。

所以如果想让 `x ^ y` 最大，就应该：

* 先让最高位尽量变成 1
* 再考虑次高位
* 再考虑更低位

这就是为什么 Trie 要按 **从高位到低位** 存。

因为这样查询时，也能按 **从高位到低位** 决策。

---

# 四、怎么查询：真正“使用”这棵树

还是用上面那三个数：

* 3 = `0011`
* 5 = `0101`
* 6 = `0110`

现在假设我们要查询：

* `x = 3 = 0011`

我们想在树里找一个数 `y`，让 `3 ^ y` 最大。

---

## 第一步：看最高位

`3 = 0011`

最高位是 `0`。

那为了让异或这一位变成 `1`，最好找什么？

当然是找 `1`，因为：

```text
0 ^ 1 = 1
0 ^ 0 = 0
```

但是树里这一层有没有 `1` 分支，要看实际存过的数。

从我们这棵树看，根下面只有 `0`，没有 `1`。

所以没办法，只能走 `0`。

---

## 第二步：看下一位

现在来到第二层，`3` 这一位还是 `0`。

还是一样：

* 最想走 `1`
* 因为 `0 ^ 1 = 1`

而这次树里真的有 `1` 分支。

所以我们就走 `1`。

注意这一步的含义：

> 我们已经确定，匹配的那个数 `y` 的这一位是 `1`。

---

## 第三步：继续往下

接下来 `3` 的第三位是 `1`。

那为了让异或这一位尽量是 `1`，最好找：

* `0`，因为 `1 ^ 0 = 1`

如果当前节点下面有 `0` 分支，就走 `0`；没有才走 `1`。

---

## 第四步：最后一位

继续同样做。

这样一路走到底，就找到一个数 `y`。

---

# 五、你真正要理解的是这句话

Trie 的“使用方式”其实就是：

> **对于 x 的每一位，优先去树里找相反位。**

因为：

* `0` 最想配 `1`
* `1` 最想配 `0`

所以代码里才会写：

```java
if (son[p][t ^ 1] != 0)
```

这里的 `t ^ 1` 就是“相反位”：

* `0 ^ 1 = 1`
* `1 ^ 1 = 0`

---

# 六、为什么“从高到低存”就能这么用？

因为树的每一层都对应一位：

* 第 1 层：最高位
* 第 2 层：次高位
* ...
* 最后一层：最低位

所以当你在第 `i` 层选分支时，本质上就是：

> **在决定匹配数 y 的第 i 位取 0 还是 1**

这就把“找最优 y”变成了“在树上走一条最优路径”。

---

# 七、我给你做一个完整的小例子

我们还是存这三个数：

* 3 = `0011`
* 5 = `0101`
* 6 = `0110`

现在查询 `x = 3 = 0011`

---

## 第 1 位：0

想找 `1`，但没有。
只能走 `0`。

---

## 第 2 位：0

想找 `1`，有。
走 `1`。

说明匹配数前两位现在是 `01xx`

---

## 第 3 位：1

想找 `0`，有。
走 `0`。

说明匹配数前三位现在是 `010x`

---

## 第 4 位：1

想找 `0`，但没有，只能走 `1`。

所以最终找到的是：

```text
0101 = 5
```

于是：

```text
3 ^ 5 = 0011 ^ 0101 = 0110 = 6
```

这就是在 Trie 里的使用过程。

---

# 八、你现在可能还会疑惑的一点

你可能在想：

> “为什么这样每一位贪心选，最后就一定是最优？”

原因是：

**异或结果比较大小时，高位优先。**

所以：

* 只要当前更高位能变成 `1`，那一定比让它变成 `0` 更优
* 低位再怎么补，也补不过高位的差距

所以这种“每一位都优先选相反位”的做法是成立的。

---

# 九、把“存”和“用”连起来看

现在你可以把整个过程连起来理解：

### 存

把每个数写成二进制，从高位到低位插入树中。
这样树的每一层都表示某一位。

### 用

查询某个数 `x` 时，也从高位到低位看。
每一位都尽量去找“相反位”，让异或结果当前位变成 `1`。

---

# 十、一句话重新解释“从最高位到最低位插入 Trie”

它的真正作用不是“单纯存起来”，而是：

> **让每一层都对应一个二进制位，这样查询时就能从高位开始贪心地找最优匹配。**

---

# 十一、你现在先记住这个最核心的画面

你可以把查询过程想成：

* 手里拿着一个数 `x`
* 从树根往下走
* 每到一层，就看 `x` 当前位是 0 还是 1
* 然后优先走相反的边
* 最后走出一个数 `y`
* 这个 `y` 就是和 `x` 异或尽量大的数

# 用什么发方式存储每一个数字，数组的大小怎么判断，请解释都详细一点
好，这个问题已经进到这题最核心的“实现层”了。

你问的是两件事：

1. **每个数字到底是怎么存进 Trie 的**
2. **数组为什么这样开、大小怎么判断**

我按这两个问题，给你彻底拆开讲。

---

# 一、先说结论：这题并没有“把每个数字单独存成一个二进制数组”

很多初学者会误以为：

* 每个数先转成二进制
* 再用一个二维数组把所有位存起来

其实这题**不是这么做的**。

这题真正的做法是：

> **每次拿到一个数时，用位运算一位一位取出它的二进制位，然后直接插入 Trie。**

也就是说：

* **数字本身**，还是正常存在 `a[i]` 里
* **Trie 里存的**，不是“第几个数字”，而是“很多数字共享出来的二进制路径”

这个区别非常重要。

---

# 二、每个数字本身是怎么存的？

这一部分最简单。

比如代码里：

```java
static int[] a = new int[N];
```

这个数组就是正常存输入的数字：

* `a[0]` 存第一个数
* `a[1]` 存第二个数
* `a[2]` 存第三个数
* ...

例如输入：

```text
3 5 6
```

那么：

```java
a[0] = 3;
a[1] = 5;
a[2] = 6;
```

这只是“原始数据存储”。

---

# 三、Trie 里存的不是“数字编号”，而是“路径”

这才是重点。

假设我们只看 4 位：

* `3 = 0011`
* `5 = 0101`
* `6 = 0110`

插入 Trie 时，不是说：

* 在某个地方直接存一个 `3`
* 再存一个 `5`
* 再存一个 `6`

而是把它们拆成路径：

* `0011`
* `0101`
* `0110`

所以 Trie 里实际存的是：

* 根节点到下一层，走 `0` 还是 `1`
* 再下一层，走 `0` 还是 `1`
* ...
* 一直走到最低位

也就是说：

> **Trie 存的是“每一位该往哪条边走”**

---

# 四、`son[p][0]` 和 `son[p][1]` 到底是什么意思？

这是最关键的一层。

代码一般写成：

```java
static int[][] son = new int[M][2];
```

你可以把它理解成：

* `son[p][0]`：编号为 `p` 的节点，它的 `0` 儿子是谁
* `son[p][1]`：编号为 `p` 的节点，它的 `1` 儿子是谁

所以每个节点最多只有两个儿子：

* 一个表示二进制位 `0`
* 一个表示二进制位 `1`

这就是 01-Trie。

---

## 举个最小例子

刚开始只有根节点，设它编号是 `0`。

如果插入数字 `5 = 0101`（只看 4 位）：

---

### 第 1 位：0

从根节点 `0` 出发，看这一位是 `0`。

如果 `son[0][0] == 0`，说明：

> 根节点还没有“往 0 走”的这条边

于是新建一个节点，比如编号 `1`：

```java
son[0][0] = 1;
```

然后走过去：

```java
p = 1;
```

---

### 第 2 位：1

现在来到节点 `1`，当前位是 `1`。

如果 `son[1][1] == 0`，就新建：

```java
son[1][1] = 2;
p = 2;
```

---

### 第 3 位：0

```java
son[2][0] = 3;
p = 3;
```

---

### 第 4 位：1

```java
son[3][1] = 4;
p = 4;
```

---

于是 `5` 就对应一条路径：

```text
0 --0--> 1 --1--> 2 --0--> 3 --1--> 4
```

这里：

* `0,1,2,3,4` 是节点编号
* 边上的 `0/1` 才是二进制位信息

所以 Trie 真正存储的是：

> **节点之间的连接关系**

而不是“我专门开一个数组保存 5 的二进制位”。

---

# 五、为什么二维数组第二维是 2？

因为这题是二进制。

每一位只有两种可能：

* `0`
* `1`

所以每个节点只需要记录两个分支：

```java
son[p][0]
son[p][1]
```

因此第二维大小就是 `2`。

---

# 六、那第一维 `M` 为什么要开这么大？

这就是你最关心的“数组大小怎么判断”。

我们一点一点推。

---

## 1. 一个数字最多会占多少层？

题目里一般是：

[
0 \le x < 2^{31}
]

说明一个数最多用 31 位表示。

也就是说：

* 最高位是第 `30` 位
* 最低位是第 `0` 位

所以插入一个数时，最多往下走 **31 层**。

---

## 2. 插入一个数，最多会新建多少个节点？

最坏情况下，这个数的每一位路径以前都没有出现过。

那它每走一层，都要新建一个节点。

一共 31 层，所以：

> **一个数最多新建 31 个节点**

---

## 3. 插入 `n` 个数，最多新建多少个节点？

最坏情况就是每个数都尽量不共用路径。

那么总节点数最多大约就是：

[
31 \times n
]

再加一个根节点。

所以通常写：

```java
static final int M = 31 * N;
```

或者保险一点：

```java
static final int M = 31 * N + 10;
```

---

# 七、为什么这样估算是对的？

因为 Trie 的节点数，本质上等于：

> **所有插入路径中，不同前缀的总数**

但最坏情况下，我们可以直接用上界估算：

* 每个数最多贡献 31 个新节点
* 共 `n` 个数
* 所以最多 `31n`

这是竞赛里很常见的数组开法。

---

# 八、举例看数组大小怎么推

假设：

```java
N = 100010
```

那么：

```java
M = 31 * N
```

大概就是：

[
31 \times 100010 \approx 3100310
]

所以：

```java
int[][] son = new int[3100310][2];
```

这样基本就够了。

---

# 九、为什么不是 `32 * N`？

因为这题数据范围是 `< 2^31`，只需要 31 位：

* 位编号从 `0` 到 `30`

一共 31 位。

如果题目允许更大的非负整数，或者你想统一按 `int` 的 32 位来写，也可以写成 `32 * N`。
但在这题里，`31 * N` 就够。

---

# 十、插入时为什么不需要额外保存“这个数字是谁”？

因为这题查询的时候，只关心：

> 当前这条路径上，某一位能不能走 0 或 1

并不需要知道：

* 这是第几个数字插进去的
* 这个数字的下标是多少

只要最后从根走到叶子，走出的那条完整路径就对应一个数。

所以这里不需要像普通字符串 Trie 那样额外记录：

* 结尾标记
* 单词个数
* 出现次数

这题只需要记录结构：

```java
son[p][0], son[p][1]
```

就够用了。

---

# 十一、完整地看一遍“每个数字怎么存进去”

假设插入 `x = 5`。

```java
int p = 0;
for (int i = 30; i >= 0; i--) {
    int t = x >> i & 1;
    if (son[p][t] == 0) son[p][t] = ++idx;
    p = son[p][t];
}
```

---

## 第一步：`p = 0`

表示从根节点开始。

---

## 第二步：取第 `i` 位

```java
int t = x >> i & 1;
```

得到当前位是 `0` 还是 `1`。

---

## 第三步：判断这条边存不存在

```java
if (son[p][t] == 0)
```

如果等于 0，说明当前节点还没有这条边。

注意这里的 `0` 不是“节点 0”，而是：

> **默认值 0 表示“没有儿子”**

因为 Java 数组初始值就是 0。

---

## 第四步：没有就新建节点

```java
son[p][t] = ++idx;
```

给这个儿子分配一个新编号。

比如原来 `idx = 7`，那么新节点编号就是 `8`。

---

## 第五步：往下走

```java
p = son[p][t];
```

走到下一层，继续处理下一位。

---

## 最后效果

当循环结束时，这个数从高位到低位对应的一整条路径就建好了。

---

# 十二、节点编号 `idx` 是干什么的？

这个也一定要懂。

因为我们用的是数组模拟 Trie，不是真的 `new Node()` 去创建树节点。

所以要靠一个变量来管理：

> **当前创建到第几个节点**

例如：

* 根节点固定用 `0`
* 第一个新节点编号 `1`
* 第二个新节点编号 `2`
* 第三个新节点编号 `3`

……

于是：

```java
static int idx = 0;
```

表示现在只有根节点。

每次新建节点时：

```java
++idx
```

这样就得到了新的节点编号。

---

# 十三、为什么根节点可以用 0？

因为我们约定：

* 根节点编号就是 `0`
* `son[p][0] == 0` 或 `son[p][1] == 0` 表示“这个儿子不存在”

这看上去好像会冲突，但实际上不会。

因为：

* “节点编号 0”只作为根节点本身使用
* 某个节点的儿子如果写成 0，表示没有儿子
* 真正新建出来的节点编号从 1 开始

所以没有问题。

---

# 十四、这题一共用了哪几种存储？

你可以整体这样看：

---

## 1. `a[]`

存原始输入数字

```java
int[] a = new int[N];
```

作用：

* 保存每个输入的数
* 后面查询时还能拿出来用

---

## 2. `son[][]`

存 Trie 的树结构

```java
int[][] son = new int[M][2];
```

作用：

* 表示每个节点的 0 儿子和 1 儿子
* 完成“路径存储”

---

## 3. `idx`

记录当前用了多少个节点

```java
int idx = 0;
```

作用：

* 给新节点编号

---

# 十五、为什么不用 `boolean[][]` 或别的结构？

因为我们不仅要知道“有没有这条边”，还要知道：

> **这条边通向哪个节点**

所以不能只记录存在不存在，而要记录儿子节点编号。

因此必须用 `int`。

---

# 十六、你可以把 `son` 想成“邻接表的超简化版”

如果你学过图，可以类比一下：

* 图里一个点连很多边
* 这里一个节点只可能连两条边：`0` 和 `1`

所以：

```java
son[p][0]
son[p][1]
```

其实就是在记录：

* 从节点 `p` 出发，经过位 `0` 到哪个点
* 从节点 `p` 出发，经过位 `1` 到哪个点

---

# 十七、数组大小判断的通用方法

以后你遇到类似 Trie 题，数组大小一般都这么想：

### 第一步：一个元素最多插入多少层

比如：

* 普通字符串 Trie：单词最大长度 `L`
* 01-Trie：二进制位数 `B`

### 第二步：每插入一个元素，最坏新建多少节点

通常就是 `L` 或 `B`

### 第三步：总元素个数是 `n`

那么总节点上界：

[
n \times L
\quad 或 \quad
n \times B
]

再加一点余量。

---

# 十八、放到这题里就是

* 每个数最多 31 位
* 有 `N` 个数
* 每个数最多新建 31 个节点

所以：

```java
M = 31 * N
```

这就是数组大小的来源。

---

# 十九、一句话总结你这个问题

### 每个数字怎么存？

不是把每个数字单独存成一个二进制数组，而是把它的二进制位从高到低插入 Trie，形成一条路径。

### 数组大小怎么判断？

一个数最多 31 位，所以最多新建 31 个节点；`N` 个数最坏共 `31 * N` 个节点，因此 `son` 的第一维开 `31 * N`。

---

# 二十、我建议你现在这样记这三个变量

你只要先把下面这三个东西记清楚，这题实现就通了：

```java
int[] a;          // 原始数字
int[][] son;      // Trie 的边
int idx;          // 当前节点编号
```

其中：

* `a[i]`：第 i 个输入数
* `son[p][0/1]`：节点 p 的 0/1 儿子
* `idx`：新建节点时用的编号计数器

---

接下来最适合的是，我直接拿这题的 `insert()` 函数，给你逐行模拟一次“插入 3、5、6 时，son 数组里到底变成了什么样”。这一步会让你彻底看懂数组是怎么表示一棵树的。




```java
import java.io.*;

class Main {
    public static void main(String[] args) throws IOException {
        BufferedReader bufferedReader = new BufferedReader(new InputStreamReader(System.in));
        BufferedWriter bufferedWriter = new BufferedWriter(new OutputStreamWriter(System.out));
        String[] input = bufferedReader.readLine().split(" ");
        int n = Integer.parseInt(input[0]);
        int[] arr = new int[n];
        input = bufferedReader.readLine().split(" ");
        for (int i = 0; i < n; i++) arr[i] = Integer.parseInt(input[i]);
        //trie二维数组的第一维是31 * n，因为每个数字都可能建立一个不一样的二进制路径
        int[][] bnums = new int[31 * n + 10][2];
        //idx=0 的时候是根节点
        int idx = 1;
        int t = 0;
        for (int i : arr) {
            int cur = 0;
            for (int j = 30; j >= 0 ; j--) {
//                t = i >> j;
                //一定要&1，才能取出最后一位
                t = (i >> j) & 1;
                if (bnums[cur][t] == 0) {
                    bnums[cur][t] = idx;
                    cur = idx++;
                }
                else {
                    cur = bnums[cur][t];
                }
            }
        }
        int res = 0;
        for (int i : arr) {
            int cur = 0;
            //total实时更新异或结果，最后的total就是一个异或值
            int total = 0;
            for (int j = 30; j >= 0 ; j--) {
                //用位运算得到每一位的二进制数
                t = (i >> j) & 1;
                int want = t ^ 1;   // 想走相反位
                if (bnums[cur][want] != 0) {
                    total = total * 2 + 1;
                    cur = bnums[cur][want];
                }else{
                    total *= 2;
                    cur = bnums[cur][t];
                }
            }
            res = Math.max(res, total);

        }
        bufferedWriter.write("" + res);

        bufferedReader.close();
        bufferedWriter.close();
    }
}
```
