# AcWing 算法基础课 -- 数据结构

## AcWing 154. 滑动窗口 

`难度：简单`

### 题目描述

给定一个大小为 $n≤10^6$

的数组。

有一个大小为k的滑动窗口，它从数组的最左边移动到最右边。

您只能在窗口中看到k个数字。

每次滑动窗口向右移动一个位置。

以下是一个例子：

该数组为[1 3 -1 -3 5 3 6 7]，k为3。

```r
窗口位置 	最小值 	最大值
[1 3 -1] -3 5 3 6 7 	-1 	3
1 [3 -1 -3] 5 3 6 7 	-3 	3
1 3 [-1 -3 5] 3 6 7 	-3 	5
1 3 -1 [-3 5 3] 6 7 	-3 	5
1 3 -1 -3 [5 3 6] 7 	3 	6
1 3 -1 -3 5 [3 6 7] 	3 	7
```

您的任务是确定滑动窗口位于每个位置时，窗口中的最大值和最小值。

**输入格式**

输入包含两行。

第一行包含两个整数n和k，分别代表数组长度和滑动窗口的长度。

第二行有n个整数，代表数组的具体数值。

同行数据之间用空格隔开。

**输出格式**

输出包含两个。

第一行输出，从左至右，每个位置滑动窗口中的最小值。

第二行输出，从左至右，每个位置滑动窗口中的最大值。

```r
输入样例：

8 3
1 3 -1 -3 5 3 6 7

输出样例：

-1 -3 -3 -3 3 3
3 3 5 5 6 7
```

### Solution

```java
import java.util.*;
import java.io.*;

public class Main{
    public static void main(String[] args) throws IOException{
        BufferedReader in = new BufferedReader(new InputStreamReader(System.in));
        String[] s = in.readLine().split(" ");
        int n = Integer.parseInt(s[0]);
        int k = Integer.parseInt(s[1]);
        // 数组 a 保存原始数据
        int[] a = new int[n + 10];
        // 数组 q 保存每一个滑动窗口处理之后的严格单调队列元素的下标
        // 注意 保存的是下标
        int[] q = new int[n + 10];
        s = in.readLine().split(" ");
        for(int i = 0; i < n; i++){
            a[i] = Integer.parseInt(s[i]);
        }
        // 1. 找滑动窗口的最小值
        // 滑动窗口的队列：队尾插入，队首弹出
        // hh 代表队首，tt 代表队尾
        // 分别初始化为 0 和 -1，保证滑动窗口为空的时候 tt < hh
        int hh = 0, tt = -1;
        for(int i = 0; i < n; i++){
            // 判断是否需要弹出队首元素
            // 当窗口队列不为空 且 当前下标和队首元素下标的差值大于 k - 1
            if(hh <= tt && i - q[hh] > k - 1) hh++;
            // 从队尾开始,弹出不大于当前元素的值
            while(hh <= tt && a[q[tt]] >= a[i]) tt--;
            // 插入当前元素
            tt++;
            q[tt] = i;
            // 队首元素即是当前窗口的最大值,打印出来
            // 当窗口元素个数达到 k 个时再输出
            if(i >= k - 1) System.out.print(a[q[hh]] + " ");
        }
        System.out.println();
        // 2. 找滑动窗口的最大值
        hh = 0;
        tt = -1;
        for(int i = 0; i < n; i++){
            if(hh <= tt && i - q[hh] > k - 1)  hh++;
            while(hh <= tt && a[q[tt]] <= a[i]) tt--;
            tt++;
            q[tt] = i;
            if(i >= k - 1) System.out.print(a[q[hh]] + " ");
        }
        System.out.println();
    }
}
```

这题属于**数据结构里的单调队列**，核心考点是：

1. **滑动窗口**
2. **单调队列维护区间最值**
3. **队列里存下标，不存值**

题目和你给的题解都很标准，属于这类题的模板题。

---

## 一、这题是什么类型

这是一个很典型的：

**“长度固定为 k 的区间，窗口每次右移一格，求每个窗口的最小值 / 最大值”**

根据你上传的学习目录，它属于第二讲 **数据结构** 中的 **队列 / 单调队列** 内容。

---

## 二、先说朴素做法，再说优化做法

### 1）朴素做法

每形成一个窗口，就把窗口里的 `k` 个数扫一遍，找最小值和最大值。

比如窗口有 `n-k+1` 个，每个窗口扫 `k` 次：

* 时间复杂度：**O(nk)**

当 `n ≤ 10^6` 时，这样会非常慢，基本不可能通过。

---

### 2）优化做法

用**单调队列**维护当前窗口内“有可能成为最值”的元素。

这样每个元素：

* 最多进队一次
* 最多出队一次

所以总复杂度是：

* **O(n)**

这就是这题的关键。

---

## 三、核心思路

---

### 1）为什么队列里存的是“下标”

题解里特别强调了这一点，这很重要。

因为我们不仅要知道值是多少，还要知道它：

* 是否已经滑出窗口
* 对应的是数组哪个位置

所以队列里必须存 **下标**，然后通过 `a[q[hh]]` 取值。

例如：

```java
q[tt] = i;
```

表示把当前元素下标 `i` 放进队列。

---

### 2）求最小值时，队列要保持什么性质

求最小值时，队列要维护成：

**从队头到队尾对应的值单调递增**

也就是：

```java
a[q[hh]] <= a[q[hh+1]] <= ... <= a[q[tt]]
```

这样一来：

* 队头就是当前窗口最小值的位置
* 输出 `a[q[hh]]` 就行

---

### 3）为什么要从队尾弹出

当新元素 `a[i]` 进入时：

如果队尾对应的值 `a[q[tt]] >= a[i]`，那么队尾那个元素以后就**不可能再成为最小值**了。

因为：

* 它比当前元素大或相等
* 而当前元素位置更靠右，活得更久

所以应该把它弹掉：

```java
while(hh <= tt && a[q[tt]] >= a[i]) tt--;
```

---

### 4）为什么还要判断队头是否过期

窗口长度固定为 `k`。

当前处理到下标 `i` 时，窗口范围是：

```java
[i - k + 1, i]
```

如果队头下标 `q[hh]` 已经小于 `i-k+1`，说明它滑出窗口了，要弹出。

题解里写的是：

```java
if(hh <= tt && i - q[hh] > k - 1) hh++;
```

这和下面写法是等价的：

```java
if(hh <= tt && q[hh] < i - k + 1) hh++;
```

都表示：**队头已经不在当前窗口里了**

---

## 五、求最小值这段代码怎么理解

```java
int hh = 0, tt = -1;
for(int i = 0; i < n; i++){
    if(hh <= tt && i - q[hh] > k - 1) hh++;
    while(hh <= tt && a[q[tt]] >= a[i]) tt--;
    tt++;
    q[tt] = i;
    if(i >= k - 1) System.out.print(a[q[hh]] + " ");
}
```

---

### 1）`hh` 和 `tt` 是什么

这是用数组模拟队列时常见写法：

* `hh`：队头指针
* `tt`：队尾指针

初始化：

```java
int hh = 0, tt = -1;
```

表示此时队列为空，因为 `tt < hh`

---

### 2）第一步：删除过期元素

```java
if(hh <= tt && i - q[hh] > k - 1) hh++;
```

当前枚举到 `i`，若队头下标已经离当前太远，说明它不在窗口内了，就删掉。

例如 `k=3`，当前 `i=4`，窗口应为 `[2,3,4]`。
如果队头下标是 `1`，那已经过期，要删除。

---

### 3）第二步：维护单调性

```java
while(hh <= tt && a[q[tt]] >= a[i]) tt--;
```

这里是最核心的一步。

为了让队列保持“值递增”，如果队尾元素比当前元素大或相等，就删掉。

原因是它永远不如当前元素优：

* 更大，不适合当最小值
* 位置还更靠左，更早过期

---

### 4）第三步：把当前元素加入队尾

```java
tt++;
q[tt] = i;
```

把当前下标放进队列。

---

### 5）第四步：窗口形成后开始输出

```java
if(i >= k - 1) System.out.print(a[q[hh]] + " ");
```

只有当 `i >= k-1` 时，长度为 `k` 的窗口才真正形成。

这时队头就是当前窗口最小值。

---

## 六、求最大值为什么只改一个方向

求最大值代码：

```java
hh = 0;
tt = -1;
for(int i = 0; i < n; i++){
    if(hh <= tt && i - q[hh] > k - 1)  hh++;
    while(hh <= tt && a[q[tt]] <= a[i]) tt--;
    tt++;
    q[tt] = i;
    if(i >= k - 1) System.out.print(a[q[hh]] + " ");
}
```

和求最小值相比，只变了一处：

```java
while(hh <= tt && a[q[tt]] <= a[i]) tt--;
```

因为现在要维护的是：

**从队头到队尾对应的值单调递减**

这样：

* 队头始终最大
* 输出 `a[q[hh]]` 就是最大值

```java
// 队首元素即是当前窗口的最小值，打印出来
```

## 八、这题的关键变量分别表示什么

### `a[i]`

原数组第 `i` 个数。

### `q`

单调队列，里面存的是**下标**。

### `hh`

队头位置。

### `tt`

队尾位置。

### `q[hh]`

当前队头元素的下标。

### `a[q[hh]]`

当前窗口的最小值或最大值。

## 九、边界处理要注意什么

### 1）窗口还没形成时不能输出

```java
if(i >= k - 1)
```

这是必须的。

因为前面元素不足 `k` 个时，还不能算完整窗口。

---

### 2）相等元素时为什么也弹

最小值写的是：

```java
a[q[tt]] >= a[i]
```

最大值写的是：

```java
a[q[tt]] <= a[i]
```

这里用了 `>=` 和 `<=`，也就是**相等时也弹出**。

这样做的好处是：

* 队列更简洁
* 保留更新、更靠右的那个元素
* 因为更靠右的元素会更晚过期，更有优势

这是单调队列模板里非常常见的写法。

## 十、给你走一遍样例，帮助你真正理解

数组：

```text
1 3 -1 -3 5 3 6 7
```

`k = 3`

我们只看“最小值”的队列变化。

---

### i = 0, a[i] = 1

队列空，直接入队：

```text
q = [0]   -> 对应值 [1]
```

窗口还没形成，不输出。

---

### i = 1, a[i] = 3

先删过期：没有
维护递增队列：

* 队尾值 `1 >= 3` 吗？不是

直接入队：

```text
q = [0, 1] -> 对应值 [1, 3]
```

窗口还没形成，不输出。

---

### i = 2, a[i] = -1

先删过期：没有
维护递增：

* `3 >= -1`，弹出
* `1 >= -1`，弹出

再入队：

```text
q = [2] -> 对应值 [-1]
```

窗口形成，输出最小值 `-1`

---

### i = 3, a[i] = -3

当前窗口应是 `[1,3]`

先删过期：

* `q[hh] = 2`，没过期

维护递增：

* `-1 >= -3`，弹出

入队：

```text
q = [3] -> 对应值 [-3]
```

输出 `-3`

---

后面同理。

你会发现：
队列里保存的并不是整个窗口所有元素，而是**有资格成为最小值的候选人**。

## 十一、规范一点的 Java 写法

你这份代码可以通过，但如果从工程角度、写题习惯来说，通常会把输出先存起来，用 `StringBuilder`，避免频繁 `print`。

我给你一版更规范、适合背模板的写法：

```java
import java.io.*;

public class Main {
    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        String[] first = br.readLine().split(" ");
        int n = Integer.parseInt(first[0]);
        int k = Integer.parseInt(first[1]);

        int[] a = new int[n];
        String[] nums = br.readLine().split(" ");
        for (int i = 0; i < n; i++) {
            a[i] = Integer.parseInt(nums[i]);
        }

        int[] q = new int[n];
        StringBuilder minAns = new StringBuilder();
        StringBuilder maxAns = new StringBuilder();

        // 求最小值：维护单调递增队列
        int hh = 0, tt = -1;
        for (int i = 0; i < n; i++) {
            // 1. 删除滑出窗口的下标
            if (hh <= tt && q[hh] < i - k + 1) {
                hh++;
            }

            // 2. 维护队列单调递增
            while (hh <= tt && a[q[tt]] >= a[i]) {
                tt--;
            }

            // 3. 当前下标入队
            q[++tt] = i;

            // 4. 窗口形成后输出队头
            if (i >= k - 1) {
                minAns.append(a[q[hh]]).append(" ");
            }
        }

        // 求最大值：维护单调递减队列
        hh = 0;
        tt = -1;
        for (int i = 0; i < n; i++) {
            if (hh <= tt && q[hh] < i - k + 1) {
                hh++;
            }

            while (hh <= tt && a[q[tt]] <= a[i]) {
                tt--;
            }

            q[++tt] = i;

            if (i >= k - 1) {
                maxAns.append(a[q[hh]]).append(" ");
            }
        }

        System.out.println(minAns.toString().trim());
        System.out.println(maxAns.toString().trim());
    }
}
```

## 十二、这题你要记住的模板结构

以后遇到“固定长度滑动窗口求最值”，就套这四步：

### 求最小值模板

```java
if (q[hh] < i - k + 1) hh++;              // 删除过期元素
while (hh <= tt && a[q[tt]] >= a[i]) tt--; // 维护单调递增
q[++tt] = i;                               // 当前元素入队
if (i >= k - 1) 输出 a[q[hh]];             // 队头即最小值
```

### 求最大值模板

```java
if (q[hh] < i - k + 1) hh++;
while (hh <= tt && a[q[tt]] <= a[i]) tt--;
q[++tt] = i;
if (i >= k - 1) 输出 a[q[hh]];
```

把这几个信号合起来，就应该想到：

**“滑动窗口 + 区间最值 + 线性时间” = 单调队列**

# 核心思路没看懂

那我们先**不讲代码**，只把“核心思路”讲到你能直觉上明白。

这题你先把它想成一句人话：

**有一个长度固定为 `k` 的小窗户，在数组上从左往右滑。每滑一次，都要马上知道这个窗户里的最小值和最大值。**

比如：

```text
数组：1 3 -1 -3 5 3 6 7
k = 3
```

窗口是长度 3：

```text
[1  3 -1] -3  5  3  6  7
 1 [3 -1 -3] 5  3  6  7
 1  3 [-1 -3 5] 3  6  7
...
```

每个窗口都要找最小值、最大值。

## 一、你为什么会觉得难

因为这题难点不在“找最小值”，而在：

**窗口一直在变，但我们不能每次都重新扫一遍。**

如果每次都把窗口里的 3 个、5 个、100 个数重新看一遍，太慢了。

所以要想一个办法：

**让窗口右移时，尽量复用前面的结果。**

这就是“单调队列”的作用。

---

## 二、先只看“最小值”，别同时想最大值

我们先只想一个问题：

**怎么快速知道当前窗口最小值？**

---

## 三、最笨的方法为什么不行

例如窗口 `[3, -1, -3]`，最小值是 `-3`。

下一个窗口 `[-1, -3, 5]`，最小值还是 `-3`。

再下一个 `[-3, 5, 3]`，最小值还是 `-3`。

你会发现：

**窗口每次只变一点点：左边出去一个，右边进来一个。**

所以没必要每次都从头找。

---

## 四、核心想法：只保留“有用的人”

把窗口里的数想象成一排人，大家都在竞争“谁能当最小值”。

这时候有些人其实**早就没资格了**，可以直接淘汰。

例如现在队伍里有：

```text
3, 5
```

这时新来了一个 `2`。

那么 `3` 和 `5` 还可能当最小值吗？

都不可能了。

因为：

* `2` 比它们更小
* `2` 还站得更靠后，会活得更久

所以 `3`、`5` 都可以删掉，只留下 `2`。

这就是单调队列最关键的想法：

**新来的数如果更优秀，就把后面那些不可能再有用的数全部挤掉。**

---

## 五、为什么叫“单调队列”

因为队列里存的值会保持某种顺序。

### 求最小值时：

我们希望队列里的值从小到大排列：

```text
队头        队尾
小  ->  大
```

例如：

```text
-3, 2, 5
```

这样最前面的那个，一定就是最小值。

所以：

**队头永远是当前窗口最小值。**

---

## 六、那窗口滑动时，到底发生什么

每次处理一个新数 `a[i]`，只做 3 件事：

### 第 1 件事：把已经滑出窗口的人删掉

窗口长度固定为 `k`。

比如当前窗口只允许下标在 `[2, 4]` 里。

如果队头那个数的下标是 `1`，它已经不在窗口里了，就必须删掉。

这叫：

**删除过期元素**

---

### 第 2 件事：新来的数进来前，把比它差的都删掉

例如现在队列里（从小到大）是：

```text
-1, 3
```

新来一个 `-3`。

那 `3` 没用了，删掉。
`-1` 也没用了，删掉。
最后只留下 `-3`。

因为以后只要 `-3` 还在窗口里，`3` 和 `-1` 永远不可能成为最小值。

这叫：

**维护单调性**

---

### 第 3 件事：把新数加到队尾

因为它现在是最新进入窗口的元素。

---

## 七、你真正要抓住的只有一句话

### 求最小值时：

* 队头放当前最小值
* 队列里只留“未来可能成为最小值的人”
* 没用的人直接删掉

---

## 八、举一个完整小例子

我们只看数组：

```text
1 3 -1 -3
k = 3
```

---

### 处理 `1`

队列空，直接进：

```text
[1]
```

---

### 处理 `3`

新来的是 `3`。

它比队尾 `1` 大，所以 `1` 不能删，因为 `1` 仍然可能是最小值。

队列变成：

```text
[1, 3]
```

---

### 处理 `-1`

新来的是 `-1`。

看队尾：

* `3` 比 `-1` 大，删
* `1` 比 `-1` 大，删

最后 `-1` 进去：

```text
[-1]
```

这时第一个窗口 `[1,3,-1]` 形成了。

最小值是谁？

就是队头：`-1`

---

### 再处理 `-3`

新来 `-3`。

看队尾：

* `-1` 比 `-3` 大，删

然后 `-3` 进去：

```text
[-3]
```

现在窗口是 `[3,-1,-3]`

最小值就是队头：`-3`

---

## 九、为什么一定要存“下标”，不能只存值

这是很多人第一次学时最晕的点。

因为我们不仅要知道它是多少，还要知道：

**它还在不在窗口里。**

比如窗口已经滑过去了，一个数即使很小，但如果它已经跑出窗口，就不能再用。

所以必须存下标，才能判断它是否过期。

例如：

* 当前处理到 `i = 5`
* `k = 3`
* 当前窗口范围是 `[3, 5]`

如果队头下标是 `2`，那它已经过期了，必须删除。

所以：

**队列里存下标，是为了判断元素有没有滑出窗口。**

值只是通过下标再去数组里取：

```text
a[q[hh]]
```

意思就是：队头下标对应的值。

---

## 十、最大值其实和最小值一模一样

刚才是“找最小值”。

那“找最大值”怎么做？

完全一样，只是方向反过来。

### 求最大值时：

让队列里的值从大到小排列：

```text
大 -> 小
```

这样队头就是最大值。

新来一个更大的数时，就把后面更小的全删掉。

---

## 十一、你现在不用记代码，只记这个画面

把单调队列想成一个“擂台”：

* 新选手来了
* 把后面比它弱的全赶走
* 最前面的那个始终是当前最强（或最弱）
* 超出窗口范围的人自动退场

这题本质就是这个过程。

---

## 十二、最后我把“核心思路”压缩成最简单版

### 求最小值：

1. 队头如果过期，就删掉
2. 队尾如果比新来的大，就删掉
3. 新来的进队尾
4. 队头就是当前窗口最小值

### 求最大值：

1. 队头如果过期，就删掉
2. 队尾如果比新来的小，就删掉
3. 新来的进队尾
4. 队头就是当前窗口最大值

---

## 十三、你下一步最应该做什么

你现在不是不会这题，你是**同时看“窗口”“下标”“队列”“最值”“代码”太多东西了，所以乱了**。

最好的办法是先只练这一件事：

**我带你手动模拟一遍最小值队列变化。**

我们可以只用这组数据：

```text
1 3 -1 -3 5
k = 3
```

我一格一格画给你看，保证你能看懂。



```java
import java.io.*;

class Main {
    public static void main(String[] args) throws IOException {
        BufferedReader bufferedReader = new BufferedReader(new InputStreamReader(System.in));
        BufferedWriter bufferedWriter = new BufferedWriter(new OutputStreamWriter(System.out));
        String[] input = bufferedReader.readLine().split(" ");
        int n = Integer.parseInt(input[0]);
        int m = Integer.parseInt(input[1]);
        /*
        一个队列存下标，队列长度是m
        一个数组存原始输入值，数组长度是n

        初始化数组
        for遍历每一个元素，维护队列
            每遇到一个新的数，考虑把队尾对应的元素删去（看需要什么单调性）
            检查对头对应的下标有没有离开窗口，考虑删去
        每次维护好一个新的窗口后，队头对应的值就是最值
        最大最小值分开统计
         */

        int[] arr = new int[n];
        String[] s = bufferedReader.readLine().split(" ");
        for (int i = 0; i < n; i++) arr[i] = Integer.parseInt(s[i]);
        int[] queue = new int[n];
        int hh = 0, tt = -1;
        //队列的len不需要，因为hh = 0, tt = -1 的时候，只要hh <= tt，就一定有元素
        for (int i = 0; i < n; i++) {
            while (hh <= tt && arr[queue[tt]] >= arr[i]) tt--;
            queue[++tt] = i;
//            if (queue[tt] - queue[hh] + 1 > m) {
            //看起来像是在算“窗口长度”，但其实单调队列里的队头和队尾，不代表当前窗口的左右端点。
            //因为队列里存的不是窗口所有元素，只存了“可能成为最值的候选下标”。
            //下面才是对的
            if (i - m + 1 > queue[hh]) hh++;
            if (i >= m - 1) bufferedWriter.write(arr[queue[hh]] + " ");
        }
        bufferedWriter.write("\n");

        hh = 0;
        tt = -1;
        for (int i = 0; i < n; i++) {
            while (hh <= tt && arr[queue[tt]] <= arr[i]) tt--;
            queue[++tt] = i;
            if (i - m + 1 > queue[hh]) hh++;
            if (i >= m - 1) bufferedWriter.write(arr[queue[hh]] + " ");
        }

        bufferedReader.close();
        bufferedWriter.close();
    }

    public static void fund() {
    }
}
```
