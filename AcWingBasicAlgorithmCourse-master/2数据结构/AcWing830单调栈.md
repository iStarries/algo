# AcWing 算法基础课 -- 数据结构

## AcWing 830. 单调栈 

`难度：简单`

### 题目描述

给定一个长度为N的整数数列，输出每个数左边第一个比它小的数，如果不存在则输出-1。

**输入格式**

第一行包含整数N，表示数列长度。

第二行包含N个整数，表示整数数列。

**输出格式**

共一行，包含N个整数，其中第i个数表示第i个数的左边第一个比它小的数，如果不存在则输出-1。

```r
数据范围

1≤N≤105

1≤数列中元素≤109

输入样例：

5
3 4 2 7 5

输出样例：

-1 3 -1 2 2
```

### Solution

```java
import java.util.*;
import java.io.*;

class Main{
    static final int N = 100010;
    static int[] a = new int[N];
    static int[] stk = new int[N];
    static int hh = 0;
    public static void main(String[] args) throws IOException{
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(System.out));
        String[] s = br.readLine().split(" ");
        int n = Integer.parseInt(s[0]);
        s = br.readLine().split(" ");
        for(int i = 0; i < n; i ++) a[i] = Integer.parseInt(s[i]);
        // 输出每个数左边第一个比它小的数
        // 单调递增的栈
        for(int i = 0; i < n; i++){
            while(hh > 0 && a[stk[hh]] >= a[i]) hh--;
            if(hh == 0) bw.write("-1 ");
            else bw.write(a[stk[hh]] + " ");
            hh++;
            stk[hh] = i;
        }
        bw.close();
        br.close();
    }
}
```
# 我没看懂为什么用单调栈
你卡住的其实是这句：

> **为什么“找左边第一个比它小的数”，要用“单调栈”？**

我换一种**更直观**的讲法，不先讲代码，先讲“它为什么合理”。

---

## 一、先别想“栈”，先想你到底要维护什么

假设数组是：

```text
3 4 2 7 5
```

当你看到一个新数时，比如看到 `7`，你想知道：

> 它左边离它最近、并且比它小的数是谁？

左边明明有很多数：`3 4 2`

但真正可能成为答案的，其实不是所有数。

---

## 二、关键想法：有些数以后永远不可能再有用了

这是单调栈最核心的地方。

比如现在左边有两个数：

```text
... 6 4
```

注意：`4` 在 `6` 的右边，而且 `4 < 6`。

那请你想一想：

### 以后如果来了一个新数 `x`

要找“左边第一个比 `x` 小的数”，`6` 还有机会当答案吗？

答案是：**没有。**

因为：

* 如果 `6 < x`，那么 `4 < x` 也成立
* 而且 `4` 比 `6` 更靠近 `x`
* 所以就算 `6` 能当答案，`4` 也会先挡在前面

于是：

> **只要右边出现了一个更小的数，左边那个更大的数就彻底废了。**

这就是为什么我们可以把一些数直接删掉，而且删掉以后不会后悔。

---

## 三、所以我们只保留“还有用”的数

那什么样的数“还有用”？

答案是：从左到右看，保留下来的这些候选数，应该是**递增的**。

例如：

```text
3 4
```

这两个都还有用。

但如果来了一个 `2`：

```text
3 4 2
```

这时 `4` 没用了，弹掉。
再看 `3`，也没用了，再弹掉。
最后只剩 `2`。

因为 `2` 比它们都小，而且更靠右，以后更有资格成为别人的“左边第一个更小值”。

---

## 四、为什么偏偏是“栈”？

因为我们每次新来一个数，只会影响**最右边那一串最近加入的数**。

比如当前保留的是：

```text
2 5 8
```

现在来了个 `6`：

* `8` 没用了，删掉
* `5` 还有用，停

你会发现：

> 每次删除的，都是“末尾连续一段”

这正好符合**栈**的特点：

* 只能看栈顶
* 只能从栈顶删
* 只能从栈顶加

所以不是“因为题目是栈题所以用栈”，而是因为：

> **失效的元素总是从最后面开始一批一批失效，所以用栈最自然。**

---

## 五、为什么弹完以后，栈顶就是答案？

这个也很关键。

我们维护的是一个**单调递增栈**，并且它里面保存的是“还没废掉的候选人”。

当当前数是 `a[i]` 时：

### 第一步：把 `>= a[i]` 的都弹掉

因为它们不可能是“比它小”的答案。

### 第二步：看栈顶

此时栈顶有两层含义：

* 它比当前数小
* 它是离当前数最近的那个还没废掉的数

所以它正好就是：

> 左边第一个比当前数小的数

---

## 六、拿样例完整走一遍，你就会明白

数组：

```text
3 4 2 7 5
```

我们维护一个“递增栈”。

---

### 1）看到 3

左边没人，所以答案是 `-1`

把 3 放进去：

```text
[3]
```

---

### 2）看到 4

栈顶 3 小于 4，不弹

所以 4 左边第一个比它小的是 3

再把 4 放进去：

```text
[3, 4]
```

---

### 3）看到 2

栈顶 4 >= 2，弹掉
因为 4 已经废了

```text
[3]
```

栈顶 3 >= 2，再弹掉
因为 3 也废了

```text
[]
```

现在栈空，所以 2 左边没有更小值，答案是 `-1`

把 2 放进去：

```text
[2]
```

---

### 4）看到 7

栈顶 2 < 7，不弹

所以 7 左边第一个比它小的是 2

把 7 放进去：

```text
[2, 7]
```

---

### 5）看到 5

栈顶 7 >= 5，弹掉
因为 7 废了

```text
[2]
```

栈顶 2 < 5，停

所以 5 左边第一个比它小的是 2

把 5 放进去：

```text
[2, 5]
```

---

你会发现整个过程本质上就是：

> **把左边“不可能再当答案的人”清理掉，剩下的人里，离当前最近的那个就是答案。**

---

## 七、你真正要记住的不是“模板”，而是这个淘汰逻辑

单调栈最重要的不是背代码，而是记住这句话：

> **如果一个数右边出现了一个比它更小（或更优）的数，那么它以后就没资格再当答案了。**

所以：

* 没用的数删掉
* 有用的数留下
* 留下来的数天然有单调性
* 删除和加入都发生在末尾
* 所以用栈维护

---

## 八、为什么不是数组、队列、链表？

### 不是队列

因为我们删的不是最早的数，而是**最近加入、已经失效的数**。

### 不是普通数组暴力扫

因为每次都扫太慢，最坏会到 `O(n^2)`。

### 栈最合适

因为：

* 新元素总是从右边来
* 失效元素总在最右边一段
* 只需要处理“最后加入的那些候选人”

这就是栈的工作方式。

---

## 九、你可以把它想成“擂台淘汰”

把左边的数想成一排候选人，后来的数来挑战。

如果新来的数更小，那么它会把左边那些比它大、而且更靠左的人全部淘汰掉。

因为以后再来一个人，要找“最近且更小”的候选人时，新的这个数一定比那些被淘汰的人更有优势。

所以最后留下来的，就是一组真正有资格的候选人。

这组候选人从小到大排着，于是就形成了**单调递增栈**。

---

## 十、最后给你一个最短版结论

这题用单调栈，不是因为它长得像栈题，而是因为：

1. 我们只关心左边“还可能成为答案”的数
2. 一个数一旦被右边更小的数压住，就永远没用了
3. 这些没用的数总是从末尾连续出现
4. 所以用栈删除最方便
5. 删除后栈顶就是离当前最近的、更小的数

---

# 为什么要用另存下标，而不是只存值？


> **只要题目结果和“位置”有关，单独存值 usually 就不够了。**

更准确地说：

* **这题只输出左边第一个比它小的“值”**
  所以你现在**只存值**，完全可以。
* **一旦题目要位置、距离、左右边界**
  那你就不能只存值了，**至少要能拿到下标**。

---

## 一、当前这题为什么存值就够

这题问的是：

> 每个数左边第一个比它小的**数是多少**

例如：

```text
3 4 2 7 5
```

对于 `7`，答案是 `2`。
题目只要你输出 `2`，不关心这个 `2` 在哪。

所以你栈里直接放值：

```java
stack[++hh] = x;
```

没有问题，因为你最后输出的也正是值。

---

## 二、什么时候“只存值”就不够了

### 1. 求左边第一个比它小的**位置**

例如数组：

```text
3 4 2 7 5
```

如果题目问：

> `7` 左边第一个比它小的位置是几？

答案是 `2` 这个数所在的位置。

这时候你如果栈里只有值：

```text
[2, 7]
```

你只知道栈顶是 `2`，但你不知道这个 `2` 是第几个位置来的。

所以这时必须存：

* 下标 `i`
* 或者 `(值, 下标)` 这对信息

---

### 2. 求距离

比如题目问：

> 每个数到左边第一个比它小的数的距离是多少？

假设：

```text
3 4 2 7 5
```

`5` 左边第一个比它小的是 `2`，位置差是：

```text
4 - 2 = 2
```

这时候如果你只知道答案值是 `2`，还不够，因为你还得知道：

* 当前数的位置 `i = 4`
* 那个更小数的位置 `j = 2`

距离要算的是：

```text
i - j
```

没有 `j`，你就算不出来。

所以**距离题本质上一定要位置**，因此只存值不够。

---

### 3. 求左右边界

比如有些题会问：

> 每个数左边第一个比它小的位置
> 每个数右边第一个比它小的位置

最后用这两个边界去算区间长度、覆盖范围、最大矩形之类。

这时候你需要的是：

* 左边界下标
* 右边界下标

显然只存值完全不够。

---

## 三、核心区别：你到底要“值”还是要“位置”

你可以这样判断：

### 只存值可以的情况

题目最后只关心：

* 左边第一个比它小的**值**
* 左边第一个比它大的**值**

这种题只输出“谁”，不关心“它在哪”。

---

### 必须拿下标的情况

题目关心：

* 第一个满足条件的**位置**
* 两个位置之间的**距离**
* 某个点的**左右边界**
* 以某个点为最小值/最大值能扩展到哪里

这种题本质都依赖位置。

---

## 四、为什么大家更喜欢“存下标”

因为**存下标更通用**。

你存下标后：

```java
stk[++tt] = i;
```

想拿值时可以写：

```java
a[stk[tt]]
```

想拿位置时直接就是：

```java
stk[tt]
```

也就是说：

* 要值，可以由下标查出来
* 要位置，下标本来就有

所以“存下标”基本兼容所有变形题。

而“只存值”只能做最简单的那类。

---

## 五、举个重复值的例子，你会更容易明白

数组：

```text
2 1 2 1
```

如果题目问：

> 第 4 个数左边第一个比它小的位置是哪里？

先不说这题具体答案，关键是：

如果栈里只有值，你看到一个 `1`，你根本分不清：

* 这是前面的第 2 个位置那个 `1`
* 还是后面别的位置那个 `1`

**值相同的时候，位置信息会丢失。**

所以一旦题目涉及位置，只存值就很危险。

---

## 六、最准确的说法

所以不是：

> “变成这样就不能存值了”

而是：

> **单独存值不够用了。**

你有三种选择：

### 方案 1：只存值

适合这题这种“只输出值”的简单情况。

### 方案 2：只存下标

最常用，最推荐。

### 方案 3：存一个二元组 `(值, 下标)`

也可以，但 Java 手写起来比“只存下标”更麻烦一些。

因为数组 `a[i]` 已经能通过下标取值了，所以通常没必要再重复存值。

---

## 七、你这题现在的代码有没有问题

没有问题。

你现在这题：

```java
stack[++hh] = x;
```

存值是**可以做对的**。

只是我前面说的意思是：

> 你现在这份写法适合“原题”，但换成“求位置/距离/边界”的变形题，就最好改成存下标。

---

## 八、你可以记一句最实用的话

> **单调栈如果题目只要值，可以存值；只要题目和位置有关，就存下标。**

---

我再给你一个最短对比：

### 当前题

问的是：

```text
左边第一个比它小的数是谁？
```

所以存值够。

### 变形题

问的是：

```text
左边第一个比它小的是第几个？
距离多远？
左右能扩到哪里？
```

所以必须有下标。

---

你要是愿意，我下一条可以直接给你写一个对照版：

* **存值版单调栈**
* **存下标版单调栈**

两份代码放一起，你一眼就能看出区别。





```java
import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;

class Main {
    public static void main(String[] args) throws IOException {
        BufferedReader bufferedReader = new BufferedReader(new InputStreamReader(System.in));
        String[] s = bufferedReader.readLine().split(" ");
        int n = Integer.parseInt(s[0]);
        int[] stack = new int[n];
        int[] arr = new int[n];
        StringBuilder stringBuilder = new StringBuilder();
        int hh = -1;
        /*
        hh 是“栈内下标”
        stack[hh] 是“原数组下标”
        a[stack[hh]] 才是值
        这样分开维护可以保留原始数组的值，同时用栈记住每一个筛选出来的元素对应于原数组的下标，
        这样不仅可以应付输出值的题目，还可以应付输出位置、距离的题目
        
        栈要维护成单调栈（这一题是递增的）
         */
        s = bufferedReader.readLine().split(" ");
        for (int i = 0; i < n; i++) {
            int x = Integer.parseInt(s[i]);
            while (hh >= 0) {
                if (arr[stack[hh]] >= x) {
                    hh--;
                } else {
                    break;
                }
            }
            if (hh == -1) {
                //看栈里面有没有存放下标
                stringBuilder.append("-1 ");
            } else {
                stringBuilder.append(arr[stack[hh]] + " ");
            }
            stack[++hh] = i;
            arr[i] = x;
        }
        System.out.println(stringBuilder.toString());
    }
}
```
